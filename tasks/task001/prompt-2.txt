用户认证系统有以下问题需要修复，代码在 src/pages/auth/ 目录下。
1. 找回密码流程在用户点击"发送重置链接"后，会先渲染一个"即将跳转到登录页…"的中间屏，等 2 秒后才切回登录 tab。提示词要求第三步直接自动跳回登录页，不需要额外展示中间状态页来拉长等待时间。
2. Login.jsx 和 Register.jsx 在提交成功后分别往回调里传了 user 参数，但父组件 index.jsx 根本没接收这个参数，而是自己又从 localStorage 读了一遍用户信息。这个多余的传参应该清理掉，保持回调的调用方和实现方一致。
3. 修改密码子组件 ChangePassword 改完密码后自己直接清了 localStorage 的登录态，而 index.jsx 中的 handlePasswordChanged 回调只更新了 React 状态、没有一起清理 localStorage。如果以后复用这个回调或者改动了子组件逻辑，清登录态容易漏掉——需要让这个回调统一负责登录态的生命周期，而不是把清理逻辑分散在两个地方。
4. 注册表单的字段顺序当前是邮箱→昵称→密码→确认密码，但提示词要求的顺序是邮箱→密码→确认密码→昵称，把昵称插在邮箱和密码之间跟需求不一致，调整字段顺序即可。
5. 修复后重新运行 pnpm run test，确保所有已有测试继续通过。
