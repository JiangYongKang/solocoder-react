在当前 React 项目中实现「商品管理后台」页面，所有相关代码放在 src/pages/products/ 下。该页面是首页任务卡片「商品管理后台」点击后的二级页面，用 localStorage 做数据持久化。

1. 设计一个商品数据结构，至少包含名称、价格、分类、库存数量、上下架状态、图片和创建时间这些字段，然后基于这个结构做一个商品列表作为页面的主体区域，用户可以在列表上按名称模糊搜索、按价格/时间/库存升降序排序、按分类筛选，列表还需要做分页。
2. 支持新建商品（通过弹窗或表单录入所有字段）、编辑已有商品信息、删除单个商品。删除时要有二次确认。
3. 用户可以在列表里多选商品，然后对选中的商品执行批量上架、批量下架或批量删除，批量删除同样需要二次确认。
4. 新建和编辑商品时允许上传图片并实时预览，图片不需要上传到服务器，转成 base64 或 Data URL 存在本地数据中即可。
5. 页面初始加载时从 localStorage 读取商品数据，每次增删改后自动写回 localStorage。
6. 在 src/pages/__tests__/products/ 下编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。
7. 在 src/App.jsx 中注册该页面的路由，确保从首页点击卡片能正确跳转。同时做好响应式布局，在 375px 和 1440px 宽度下均可正常使用。
