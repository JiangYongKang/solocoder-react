任务看板在拖拽排序上有几个 BUG 需要修复，代码在 src/pages/kanban/ 目录下。
1. 拖拽排序在相同列内移动卡片时，fromIndex 是从闭包中捕获的旧 board 值算出来的，不是在 setBoard 的函数式更新里从 prev 推出来的。如果拖拽过程中正好有其他操作改动了 board 状态，fromIndex 会指向错误位置，可能删错甚至删掉不相关的卡片。把这个 fromIndex 的取值改成从函数式更新的 prev 参数推导。
2. 拖拽卡片时只有卡片本身变半透明，列内看不到任何插入位置指示。松手前完全不知道卡片会落在哪两个卡片之间，拖拽操作基本没法准确使用。需要加上落点视觉指示，比如在目标插入位置显示一条插入线或留个间隙。
3. getDropIndex 函数里面直接查 querySelectorAll('.kanban-card-slot') 去拿 DOM 节点算位置索引，把拖拽逻辑跟样式类名绑死了。类名一改整个拖拽排序直接挂。改成不依赖 CSS 类名的方式来确定插入位置索引。
4. 用 pnpm run dev 在浏览器里实际验证拖拽行为，以及 375px 宽度下页面能不能正常使用。跑完 pnpm run test 确认所有已有测试通过。
