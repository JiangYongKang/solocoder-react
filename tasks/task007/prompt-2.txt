表单构建器有以下问题需要优化，代码在 src/pages/form-builder/ 目录下。
1. addOption 函数在算新选项编号时用了 field.options.length + 1，但如果用户先删掉了中间的某个选项再点添加，就会搞出重复的选项值。比如原来有三个选项，用户把第二个删了再添加，两个选项都会标成 option3。改成递增不重复的方式来生成选项编号。
2. 字段拖拽到画布的体验可以更友好一些——现在画布上只有一两个字段的时候，字段之间的缝隙太小了，想把一个新字段精确拖到两个已有字段之间很费力。在字段之间加一条可见的放置指示线，落到这个线上代表插在两个字段之间。
3. 导出 JSON Schema 时 required 这一个字段表现不一致：空画布导出时不写 required 字段，但画布有字段却全部不是必填时反而导出了一个空数组 []。没有必填字段的时候两种场景按同一个规则处理，别一个不写一个写空数组。
4. 属性面板里数字输入框的 placeholder 被直接隐藏了，实际上 HTML 的 type="number" 输入框是支持 placeholder 属性的。提示词只说数字输入框需要设最大最小值，没说不让用 placeholder。把 placeholder 属性正常传递下去，别吞掉。
5. CSS 里的响应式断点设了 1024px、768px 和 1440px，偏偏没有提示词明确要求的 375px。给超窄屏补上对应的样式适配，至少让字段面板和画布在 375px 宽度下不挤不乱。
6. 修复后跑 pnpm run test，所有已有测试需要继续保持通过。
