产物不满意：
1. Alert、Badge 和输入框聚焦环的 CSS 中把颜色硬编码成了 rgba 值，比如 .te-alert-info 的 background 写死了 rgba(170, 59, 255, 0.1)，完全不走 CSS 变量。用户改主色后这些组件的背景色和边框色纹丝不动，右侧预览没做到"样式完全由左侧控件驱动"。
2. 颜色编辑器的 HEX 文本输入框没有任何输入校验，随便敲几个字母就能直接写进 config 里，CSS 变量被设成无效值，预览区可能直接展示异常。
3. 亮色/暗色模式切换时用户调过的自定义颜色会全部丢失。setMode 里用 getColorsByMode 返回的完整默认配色直接盖掉了用户之前手动修改的颜色，从亮色切到暗色再切回来，之前在亮色下调过的颜色就没了。
4. 卡片 hover 的 box-shadow 硬编码为 rgba(0, 0, 0, 0.1)，暗色模式下浅色阴影叠在深色背景上看着不协调，没有根据主题模式适配阴影颜色。

过程不满意：
1. 模型收到提示词后第一件事就是建 TodoList，然后才开始读项目文件，违反了"应先分析问题再规划"的原则。
2. 修改 App.css 时编辑操作出错导致整个文件被破坏，不得不整体重写，说明工具使用不够精准。
3. 首次运行 pnpm test 时直接报错 localStorage 未定义，模型没有先检查 vitest 配置或项目已有的测试 mock 方式就直接去翻其他测试文件参考，属于试错式调试而非先分析再动手。
