主题定制系统有以下 BUG 需要修复，代码在 src/pages/theme-editor/ 目录下。
1. Alert 提示框、Badge 标签和输入框聚焦环的背景色和边框色在 CSS 里被写死成了 rgba 常量，比如 .te-alert-info 的 background 是 rgba(170, 59, 255, 0.1)。这意味着用户在左侧面板改了主题主色之后，这些组件的颜色根本不跟着变，右侧预览并没有做到"样式完全由左侧控件驱动"。把这些硬编码的颜色值改成引用 CSS 变量，让它们跟随主题配置动态变化。
2. 颜色编辑器的 HEX 文本框现在什么东西都能往里敲，敲几个乱码直接就写进 config 了，CSS 变量被设成无效值之后整个预览区可能直接崩掉。给 HEX 输入加上基本的格式校验，不合法的值不要让它写进 config。
3. 亮色模式和暗色模式之间切换的时候，用户之前手动调过的自定义颜色会全部丢光。原因是 setMode 里调了 getColorsByMode 直接拿了一套完整默认配色盖掉了当前 config，之前在亮色下辛辛苦苦调的颜色切到暗色再回来就全没了。模式切换时应该只替换那些用户没改过的默认色，手动调整过的颜色要保留。
4. 卡片 hover 时的 box-shadow 写死了 rgba(0, 0, 0, 0.1)，在暗色模式下这个浅色阴影叠在深色背景上看着很别扭。让阴影颜色也根据当前模式适配。
5. 修完跑 pnpm run test 确保已有测试全部通过。
