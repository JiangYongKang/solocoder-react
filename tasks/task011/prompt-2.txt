当前项目 src/pages/calendar/ 下的日程日历页面存在一些问题，请在该目录下修复它们。

1. 日视图和周视图中的事件拖拽调整起止时间功能完全无效。按住事件上下移动或拖拽底部手柄时，事件位置不跟随鼠标，松手后也不会更新。需要修复拖拽的闭环机制，确保 handleMove 里能正确拿到最新的拖拽状态和事件引用。
2. 拖拽的位移换算比例和事件渲染定位比例不一致——视图渲染用 60px 表示一小时，但拖拽计算改用了 1.2px/min 的比例。同一段鼠标移动距离，视图上看到的位移量和实际调整的时间量不对应，松手后事件会跳到意料之外的位置。
3. 跨日事件在日视图中显示位置不对。比如一个昨晚 22:00 开始、今天凌晨 2:00 结束的事件，在今天的日视图中 top 值会远远超出可视区域，用户完全看不到这条事件。
4. WeekView 中水平拖拽事件切换日期时，当前用硬编码的 180px 作为一天列宽去计算 deltaDay，但周视图列宽度是 flex 自适应的，移动端和桌面端的单列宽度差异很大，同样的鼠标距离在不同屏幕下会跳不同天数。
5. snapTimeToSlot 工具函数有一个边界 BUG：当分钟数四舍五入后等于 60（比如 9:53 进到 10:00），setMinutes(60) 后小时已经自动进位，接着又 setHours 加了一次，导致输出多了一小时。
6. 修复后需要为该页面补充纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。测试代码放在 src/pages/__tests__/calendar/ 下。
7. 确保所有子页面在 375px 移动端宽度下布局不崩坏，使用 pnpm run dev 启动后手动验证关键页面。
8. 重新运行 pnpm run test，需要让所有测试通过。
