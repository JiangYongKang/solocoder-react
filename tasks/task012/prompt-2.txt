当前项目 src/pages/orders/ 下的电商订单系统页面存在一些问题，请在该目录下修复它们。

1. 购物车面板中修改商品数量时没有把库存上限传进去，导致购物车内增加数量不受库存约束。但在商品列表中点"加入购物车"时是有库存约束的，同一场景两条路径行为不一致。
2. utils.js 里定义了一个 getProductById 函数，同时也引入了 PRODUCTS 常量，但这两个东西在 orders 目录下没有任何组件实际使用，属于残留的无效代码，需要清理。
3. 订单列表按状态分 tab 展示时少了"已取消"这个 tab。取消后的订单只能在"全部"里看到，而待付款、已付款、已发货、已完成各自都有独立 tab，筛选维度不完整。
4. 商品图片是直接指向外部 API 生成的，而且 img 标签没有 onError 兜底。一旦外部服务不可用，所有商品的图片位都会破裂，应该做一个 fallback 处理。
5. 修复后需要为该页面补充纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。测试代码放在 src/pages/__tests__/orders/ 下。
6. 确保所有子页面在 375px 移动端宽度下布局不崩坏，使用 pnpm run dev 启动后手动验证关键页面。
7. 重新运行 pnpm run test，需要让所有测试通过。
