当前项目 src/pages/map-area/ 下的交互式地图页面存在一些问题，请在该目录下修复它们。

1. App.jsx 中没有注册 map-area 的路由，首页也没有对应的任务卡片，从首页完全无法跳转到这个页面。需要补上 import、任务卡片和 Route。
2. 地图上标记点的聚类算法有缺陷：目前以每组第一个点作为距离基准去判断其他点是否应聚合，但如果 A 靠近 B、B 靠近 C 且 A 与 C 距离超出阈值，C 不会被纳入同一簇。这种链式分布的标记点应该被识别为同一组。
3. 地图画布上每次拖拽平移结束后，紧接着的第一次单击会被吞掉。原因是 mouseUp 里把 isDragging 置为 false 是异步 state 更新，紧随的 click 事件内读到的是旧值 true，导致 click 逻辑被跳过。
4. 两个标记点之间的路径规划每次绘制出来的路线都不同——即使选中同样的起点和终点，切换回来后再触发路线，折线的中间转向点位置都会变，路线缺乏确定性。
5. 搜索框输入关键字如果没有匹配到预设地点，当前会在地图中心随机位置添加一个标记点。这会让用户困惑，不如无匹配时直接提示无结果。
6. 修复后需要为该页面补充纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。测试代码放在 src/pages/__tests__/map-area/ 下。
7. 确保所有子页面在 375px 移动端宽度下布局不崩坏，使用 pnpm run dev 启动后手动验证关键页面。
8. 重新运行 pnpm run test，需要让所有测试通过。
