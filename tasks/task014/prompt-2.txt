当前项目 src/pages/rich-editor/ 下的富文本编辑器页面存在一些问题，请在该目录下修复它们。

1. App.jsx 中没有注册 rich-editor 的路由，首页也没有对应的任务卡片，从首页完全无法跳转到这个页面。需要补上 import、任务卡片和 Route。
2. 撤销和重做之后，textarea 的光标会跳回到文档最开头，而不是停留在操作发生时的位置。而正常编辑（比如点工具栏按钮插入格式）时光标是能保持的，撤销重做和普通编辑的光标行为不一致。
3. 键盘打字时每敲一个字符就推一条历史记录进操作栈，撤销时逐字回退，100 条上限敲几行字就满了，撤销粒度太细。撤销应该按逻辑单元操作（比如一段连续输入、一次工具栏格式操作），而不是以单个字符为单位。
4. 提示词要求支持"粘贴 URL"插入图片，目前只有弹窗手动输入和 FileReader 上传两种方式。用户把图片链接 Ctrl+V 直接粘贴到编辑区时只会得到纯文本 URL，不会被自动转换成 Markdown 图片语法，缺少 onPaste 事件处理。
5. DEFAULT_CONTENT 的示例内容里，下划线效果用的是 HTML 标签 `<u>下划线</u>` 而非 Markdown 语法，和其他格式示例的写法不统一，用户看了会疑惑到底该用哪种方式。
6. 修复后需要为该页面补充纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。测试代码放在 src/pages/__tests__/rich-editor/ 下。
7. 确保所有子页面在 375px 移动端宽度下布局不崩坏，使用 pnpm run dev 启动后手动验证关键页面。
8. 重新运行 pnpm run test，需要让所有测试通过。
