当前项目 src/pages/rich-editor/ 下的富文本编辑器页面有一个遗留问题，请在该目录下修复它。

1. handlePaste 中通过 FileReader 异步读取粘贴的图片文件时，读取的回调里用了闭包捕获的 content、start、end 去计算插入位置。但 fileToBase64 是异步的，等待期间用户可能继续编辑导致闭包中的 content 已经过时，最终图片会插错位置甚至覆盖掉新输入的文字。需要确保粘贴图片时使用的是插入时刻的最新内容状态。
2. 用户 Ctrl+V 粘贴普通文本时，handlePaste 对 text 类型的处理没有调用 commitPendingInput，导致粘贴内容和粘贴前的连续输入被合并到同一个历史单元里。而工具栏和快捷键操作都会先提交待合并的输入，粘贴 text 的行为需要跟它们对齐。
3. 确保在 375px 移动端宽度下编辑器布局不崩坏，使用 pnpm run dev 启动后实际调整视口宽度手动验证左右分栏和工具栏的可用性。
4. 重新运行 pnpm run test，需要让所有测试通过。
