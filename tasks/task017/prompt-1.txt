在当前 React 项目中实现「流程编排器」页面，所有相关代码放在 src/pages/workflow/ 下。该页面是首页任务卡片「流程编排器」点击后的二级页面，用 localStorage 做数据持久化。

你需要构建一个可视化的流程图编辑器，用户从零搭建流程、连接节点、配置属性，最后还能模拟跑一遍看看流转路径。

1. 左侧是节点类型面板，包含开始节点、结束节点、任务节点、条件分支节点和并行网关节点五种类型。从左侧拖拽节点到中间画布即可添加一个节点。
2. 画布支持缩放和拖拽平移。节点之间可以拖拽连线——鼠标从一个节点的输出锚点拖到另一个节点的输入锚点，用贝塞尔曲线绘制连线箭头。
3. 点击画布中的某个节点，右侧弹出属性编辑面板，不同节点类型有不同的可配项：任务节点可编辑名称和处理人，条件节点可编辑判断表达式，等等。属性修改后实时更新到节点上。
4. 整个流程图的数据用 JSON 结构表达，支持导出 JSON 文件下载和从 JSON 文件导入还原。刷新页面后画布状态从 localStorage 恢复。
5. 提供一个「模拟执行」按钮，从开始节点出发，按连线方向逐步高亮经过的节点和路径，条件分支随机选一条路径走。执行过程有动画效果。
6. 在 src/pages/__tests__/workflow/ 下编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。
7. 在 src/App.jsx 中注册该页面的路由，确保从首页点击卡片能正确跳转。同时做好响应式布局，在 375px 和 1440px 宽度下均可正常使用。
