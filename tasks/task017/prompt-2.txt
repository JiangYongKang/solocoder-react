当前项目 src/pages/workflow/ 下的流程编排器页面存在一些问题，请在该目录下修复它们。

1. 并行网关的模拟执行有逻辑错误。当前代码把并行网关的所有出边都加进了高亮列表，但实际上只沿着第一条出边继续遍历后面的节点，其余并行分支上的节点根本没有被走到，后半段流程缺失了。
2. 画布组件的 handleDeleteNode 手写了一版数组过滤去删节点和连线，而 index.jsx 中同样逻辑用的是 workflowCore 里的 deleteNode 和 deleteEdgesByNodeId。同一个删除操作有两个不同的实现，而且画布版没有经过核心工具层的校验，行为可能不一致。
3. 导入 JSON 还原流程图时，对 node 做了 id、type、坐标的完整校验，但 edge 只检查了 source/target 引用的节点存不存在，没有要求 edge 自身带上有效的 id 字段。导入一个没有 id 的边数据后渲染会出问题。
4. 工具栏的放大/缩小按钮用 1.2 倍作为步进，而鼠标滚轮缩放用 0.9/1.1 倍，同一个画布上两种缩放方式的行为不一致，操作体验割裂。
5. 修复后需要为该页面补充纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。测试代码放在 src/pages/__tests__/workflow/ 下。
6. 确保所有子页面在 375px 移动端宽度下布局不崩坏，使用 pnpm run dev 启动后手动验证关键页面。
7. 重新运行 pnpm run test，需要让所有测试通过。
