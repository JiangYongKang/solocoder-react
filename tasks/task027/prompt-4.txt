你实现的代码片段管理器页面在第三轮修复后还存在一个问题，需要进行修复：

1. renderMarkdown 函数的 sanitizeUrl 在处理链接 URL 时存在二次转义问题。当前 mdProtectInline 接收的入参已经是经过 mdEscapeHtml 全量转义后的内容，链接正则从这段已转义文本中捕获到的 URL（linkUrl）本身已包含 HTML 实体（如 &lt;），sanitizeUrl 判断 URL 安全后又对这个 linkUrl 再调一次 mdEscapeHtml，导致 URL 中 HTML 实体里的 & 字符被再次转义成 &amp;amp;lt;。以 Markdown `[text](https://example.com?q=<x>)` 为例，mdEscapeHtml 先将尖括号转为 &lt; 和 &gt;，链接正则匹配到的 URL 变为 `https://example.com?q=&lt;x&gt;`，sanitizeUrl 再次转义后变成 `https://example.com?q=&amp;lt;x&amp;gt;`，最终生成的链接目标与用户原始输入不一致。需要修复这个 URL 二次转义问题，确保链接 URL 语义正确。
2. 修复后重新运行 pnpm run test，确保所有测试通过。确保所有子页面在 375px 和 1440px 宽度下布局不崩坏，需要启动 dev server 实际验证。
