3. 测试文件 pomodoroUtils.test.js 中 sendNotification 的测试用例"权限已授权时成功发送通知"（约第 607-617 行）创建了 mockNotification = vi.fn() 替换全局 Notification 构造器，但测试中只断言了 result 为 true，没有对 mockNotification 做 toHaveBeenCalled() 断言来验证 Notification 构造函数确实被调用。该测试用例的名称宣称"成功发送通知"，但实际上它只验证了代码没抛异常，即使 sendNotification 函数内部直接 return true 而不构造 Notification 对象，该测试也会通过。根本原因是 mock 设置完成后未利用 vi.fn() 的调用追踪能力验证副作用。正确做法是在测试末尾加上 expect(mockNotification).toHaveBeenCalled() 或 expect(mockNotification).toHaveBeenCalledWith('测试标题', { body: '测试内容' })。
