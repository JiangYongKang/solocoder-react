在当前项目中实现 API 调试工具，该页面是首页任务卡片"API 调试工具"点击后的二级页面。

这个页面是一个类似 Postman 的前端 API 调试工具，用户可以在浏览器里构造 HTTP 请求、发送并查看响应。

1. 请求配置区：提供 HTTP 方法下拉选择（GET、POST、PUT、PATCH、DELETE），URL 输入框，以及发送按钮。URL 输入框旁边有一个请求参数区域，以键值对表格的形式编辑 Query Parameters，支持添加行、删除行、启用/禁用某一行。参数表格中的 key 和 value 可以自由编辑。
2. 请求头编辑区：以键值对表格的形式编辑 Request Headers，同样支持添加、删除、启用/禁用。预设几组常用 Content-Type（如 application/json、application/x-www-form-urlencoded），点击预设自动填充到请求头表格中。
3. 请求体区域：当请求方法为 POST、PUT、PATCH 时显示。提供一个文本编辑区用于输入 JSON 格式的请求体，编辑区旁边提供 JSON 格式化按钮（缩进美化）和压缩按钮（去除空白）。
4. 响应展示区：发送请求后显示响应结果，包含状态码（用不同颜色区分 2xx 绿色、3xx 蓝色、4xx 橙色、5xx 红色）、响应时间（毫秒）、响应体大小。响应体用语法高亮的只读代码区展示，提供 JSON 格式化美化显示，同时显示响应头列表。
5. 历史记录：每次发送请求后自动保存记录（URL、方法、时间），左侧显示历史列表，点击历史记录可以回填请求配置。支持收藏历史记录（星标），收藏的排在列表顶部，支持删除单条或清空全部历史。
6. 环境变量管理：提供一个环境变量面板，用户可以用键值对定义多组环境变量（如开发环境、生产环境）。在 URL 和请求参数中可以用 {{变量名}} 引用环境变量，发送请求时自动替换为实际值。支持切换当前环境、新增和删除环境组。
7. 代码实现需要放在 src/pages/api-debugger/ 下，测试代码放在 src/pages/__tests__/api-debugger/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。
