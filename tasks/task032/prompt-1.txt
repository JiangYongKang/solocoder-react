在当前项目中实现 Git 仓库浏览器，该页面是首页任务卡片"Git 仓库浏览器"点击后的二级页面，全部使用模拟数据，不依赖真实 Git 仓库。

这个页面模拟一个 Git 仓库的可视化管理工具，用户可以在浏览器里查看仓库结构、变更状态和历史记录。

1. 文件树侧边栏：左侧展示项目文件树，模拟一个多层级目录结构（至少 3-4 级深度，20+ 个文件）。每个文件/文件夹前面用颜色图标标记变更状态：绿色 + 表示新增、红色 - 表示删除、橙色 ~ 表示修改、灰色表示未变更。点击文件夹可以展开折叠，点击文件在右侧打开文件内容。
2. 文件内容与 Diff 对比区：右侧主区域显示当前选中文件的内容。文件有变更时，用一个切换按钮在「工作区内容」和「变更 Diff」之间切换。Diff 视图采用左右分栏或统一 diff 模式展示修改前后的差异行，新增行绿色背景、删除行红色背景。
3. 提交历史时间线：底部或右侧面板展示当前分支的提交历史，用时间线样式渲染（竖线串联的节点）。每条提交显示提交哈希（短 7 位）、作者、时间、提交信息。点击某条提交后，文件树和内容区切换到该提交时的状态。
4. 分支切换：顶部提供一个分支下拉选择器，列出模拟的几个分支（如 main、develop、feature/login、bugfix/header）。切换分支后，文件树和提交历史刷新为对应分支的数据，当前分支名高亮显示。
5. Stage/Unstage 面板：提供一个变更文件列表，分为「已暂存」和「未暂存」两个区域。用户可以将未暂存区的文件拖入或点击移入已暂存区（模拟 git add），也可以从已暂存区移出（模拟 git reset）。所有操作都是前端模拟，不做真实的 Git 操作。
6. 代码实现需要放在 src/pages/git-browser/ 下，测试代码放在 src/pages/__tests__/git-browser/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。
