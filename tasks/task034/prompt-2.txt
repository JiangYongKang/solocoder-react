你实现的音视频播放器页面还存在一些问题，需要进行修复：

1. 播放列表缺少"全部添加到队列"的功能入口和实现逻辑，目前只有清空按钮。
2. 双击视频画面切换全屏功能与单击播放/暂停存在冲突。在同一个元素上同时绑定了 onClick 和 onDoubleClick 事件，双击时会先触发两次单击导致播放状态被反复切换，影响正常使用。
3. 断点续播的 localStorage 保存频率过高，currentTime 每次更新都触发写入，同时还有一个每 5 秒的定时保存，形成了双重冗余。需要降低写入频率。
4. 全屏模式下控制栏隐藏后（opacity: 0），控制栏区域仍然可以响应点击操作，用户可能误触到隐藏的按钮。
5. 进度条拖拽过程中，mousemove 事件处理函数里每次都用 querySelector 查询 DOM 节点，拖拽时高频执行影响性能。
6. 修复后需要补充纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。测试代码放在 src/pages/__tests__/media-player/ 下。
7. 修复完成后启动 pnpm run dev，手动验证上述交互问题是否已解决。
8. 重新运行 pnpm run test，需要让所有测试通过。
