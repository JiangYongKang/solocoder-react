你实现的 SKU 规格选择器页面还存在一些问题，需要进行修复：

1. 预览区初始状态下所有规格值按钮都被禁用（灰色不可点击），用户无法开始选择。getDisabledValues 函数在无前置选择时不应该禁用任何按钮，只有用户实际选中某个规格值后才应联动禁用无库存的组合。

2. 预览区的图片展示将多张匹配图片通过绝对定位完全重叠在一起，用户看不清任何一张图片的内容。另外，用户在某个规格维度选择了未上传图片的规格值时，之前已显示的图片会消失，应该保持上一张有效图片继续显示。

3. SkuListPanel 中，用户清空 stock 或 price 输入框时，数据模型中的对应字段被赋值为空字符串 ''，与其他场景下默认生成的数字 0 类型不一致。

4. 修改规格组名称或规格值名称（但不改变 ID 映射关系）时，预览区中用户已做的所有规格选择会被清空重置。仅改变名称信息不应触发选择重置。

5. 修复后需要为该页面补充纯 JS 函数的单元测试，覆盖上述修复涉及的边界场景。测试代码放在 src/pages/__tests__/sku-selector/ 下。

6. 重新运行 pnpm run test 和 pnpm run lint，确保测试通过且无 ESLint 报错。
