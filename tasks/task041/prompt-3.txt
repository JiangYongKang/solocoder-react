你实现的 SKU 规格选择器页面经过第二轮修复后，仍存在以下残留问题需要进行修复：

1. 当用户在 SKU 编辑面板中删除某个规格组时，该规格组在 imageHistory 中关联的图片缓存没有被同步清理。如果用户曾给该规格组的某个规格值配置过预览图，删除规格组后预览区仍会持续显示该已被删除的规格组的图片，且无法通过任何 UI 操作清除。需要确保删除规格组时同步清理对应的图片缓存。

2. 当用户点击"清空选择"按钮清空所有规格值选择后，PreviewPanel 预览区没有回到"暂无商品图片"的初始占位状态，仍然显示上一轮缓存的图片。清空选择是一个明确的用户意图，应同时清空图片缓存并回到占位状态。

3. 修复后需要确认已有的单元测试仍然全部通过，如有失败需修正。测试代码在 src/pages/__tests__/sku-selector/ 下。

4. 重新运行 pnpm run test 和 pnpm run lint，确保测试通过且无 ESLint 报错。修复完成后启动 pnpm run dev 验证页面可正常访问。
