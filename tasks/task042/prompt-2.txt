你实现的图片编辑器页面还存在一些问题，需要进行修复：

1. 滤镜调节滑块变化时不会记录到撤销历史中，撤销操作只能回退裁剪、旋转等操作，无法回退滤镜修改。提示词要求记录每一步编辑操作，滤镜修改应纳入撤销/重做系统。
2. 模糊滤镜的实现直接使用了浏览器的 CSS filter（ctx.filter = "blur(...)"），而非像其他四个滤镜那样通过像素级别操作 imageData。需要与其他滤镜保持一致，使用像素级算法实现模糊效果。
3. 导出函数 doExport 中，当滤镜为默认值时存在一段永远不会执行的死代码分支（areFiltersDefault 为 true 时 blur 必为 0，但后续又判断了 filters.blur > 0）。
4. 左侧缩略图面板中图片格式的检测逻辑过于粗糙，仅通过 data URL 字符串是否包含 "jpeg" 来判断格式，上传 WebP 图片时会被错误识别为 PNG。
5. 修复后需要确认单元测试仍然全部通过，如有失败需修正。测试代码在 src/pages/__tests__/image-editor/ 下。
6. 重新运行 pnpm run test 和 pnpm run lint，确保测试通过且无 ESLint 报错。修复完成后启动 pnpm run dev 验证页面可正常访问。
