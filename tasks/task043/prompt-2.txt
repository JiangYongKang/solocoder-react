你实现的多语言国际化管理页面还存在一些问题，需要进行修复：

1. 树形视图模式下叶子节点的行缺少删除按钮。平铺视图的每行都有操作列（包含删除按钮），但树形视图的 TreeRow 组件没有渲染操作列。同时表头的"操作"列在树形视图下与 tbody 列数不对齐。

2. 树形视图模式下叶子节点的 key 名称不可编辑。TreeRow 将 key 渲染为普通文本标签，不支持双击进入编辑状态，而平铺视图的 KeyEditCell 组件支持这一功能。两种视图模式应提供一致的行操作能力。

3. KeyEditCell 组件定义了 onChange prop，但父组件在渲染时从未传递该 prop，导致 onChange 回调代码始终不会被调用。这是一段死代码，应该清理。

4. 树形视图的初始展开 key 集合硬编码为默认数据中的顶级命名空间（如 app_0、user_0 等），用户导入其他命名空间的数据后，新的顶级节点默认处于收起状态。展开集合应根据实际数据的顶级 key 列表动态初始化。

5. 修复后需要为该页面补充单元测试，覆盖两种视图模式下的行操作场景。测试代码放在 src/pages/__tests__/i18n-manager/ 下。

6. 重新运行 pnpm run test 和 pnpm run lint，确保测试通过且无 ESLint 报错。
