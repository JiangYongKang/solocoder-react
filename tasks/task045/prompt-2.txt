你实现的 PDF 阅读器页面还存在一些问题，需要进行修复：

1. 内置模拟文档 mockDocument.js 只包含 18 页，未达到提示词要求的约 20-30 页。同时多页文本字数远低于要求，如第 18 页仅 162 字、第 4 页 348 字，而提示词要求每页约 500-800 字。需要扩充文档的页数和每页字数。

2. 使用内置模拟文档时，右侧目录面板始终显示"暂无目录"。原因是 chapterTitle 被存在页面的元数据字段中，但目录提取函数只扫描 content 正文内容，读取不到章节标题。使用内置文档时自动目录生成功能完全失效。

3. 文本搜索功能始终从第 1 页开始检索，而非从用户当前阅读的页码开始。当用户在第 15 页阅读时搜索某个词，结果会直接跳回第 1 页的匹配处，打断了阅读上下文。

4. 修复后需要确认单元测试仍然全部通过，如有失败需修正。测试代码在 src/pages/__tests__/pdf-reader/ 下。

5. 重新运行 pnpm run test 和 pnpm run lint，确保测试通过且无 ESLint 报错。修复完成后启动 pnpm run dev 验证页面可正常访问。
