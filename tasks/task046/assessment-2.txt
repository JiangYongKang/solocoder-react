2. 提示词第3条明确要求"重新运行 pnpm run test 和 pnpm run lint，确保测试通过且无 ESLint 报错"，即要求运行全量项目测试和全量项目 lint。但模型实际执行的命令是 pnpm run test -- src/pages/__tests__/flash-sale/ 和 pnpm exec eslint src/pages/flash-sale/，分别将测试和 lint 限定在了 flash-sale 目录下，没有按照提示词给出的完整命令执行。项目的 test 脚本定义为 vitest --run（全量运行），加上路径过滤参数后只跑了 flash-sale 目录的测试用例，无法验证此次边界修改是否影响了其他模块的测试。根本原因是模型在处理运行指令时自行添加了路径限制，未严格遵守提示词的要求。正确做法是直接执行 pnpm run test（不加额外参数）和 pnpm run lint，确保全项目验证通过。
