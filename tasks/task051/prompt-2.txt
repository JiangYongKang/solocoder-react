你实现的物流轨迹追踪页面还存在一些问题，需要进行修复：

1. 签收人信息不完整：除顺丰外，圆通、中通、韵达、京东四种快递的物流数据解析函数中 signer 字段都硬编码为 null，没有从各自的数据结构中提取签收人信息。需要为每种快递格式正确提取签收人字段。
2. 地图终点渲染 BUG：extractRoutePoints 函数在处理轨迹点时，如果目的地城市名称也出现在物流节点的 location 字段中，会导致终点节点被普通途经点覆盖，地图上无法正确显示「终」或签收图标。需要确保目的地标记不会被途径点覆盖。
3. 异常状态判断不可靠：顺丰物流状态的 default 兜底分支通过检测 remark 字符串中是否包含"异常"中文来判定异常，而不是依赖数据结构中的 opcode 值或异常标识字段。如果备注文字发生变化但 opcode 仍是异常码，状态就会被错误映射。需要改为基于数据结构字段驱动而非文本内容匹配的判断方式。
4. 工具函数接口被测试绑架：addQueryRecord、loadQueryHistory、saveQueryHistory 等函数为了配合测试而额外增加了 storage 参数来接收 localStorage 对象，这改变了生产代码的函数签名。应该在生产代码中直接使用 window.localStorage，在测试侧通过 mock 全局对象来解决。
5. 重新运行 pnpm run test，确保所有测试通过。
