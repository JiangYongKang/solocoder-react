产物不满意：
1. 互换按钮未同步调换换算金额。index.jsx 的 handleSwap 只交换了货币代码，但 ConversionPanel 内部的 baseAmount 和 targetAmount 是本地 state，互换后金额未变动。提示词第 2 条要求"换算金额同步调换"，点击互换后页面仍显示原金额而非将原结果作为基准金额重新换算。正确做法是在 swap 时通过回调或 key 重置让 ConversionPanel 内部金额同步交换，或加 useEffect 监听货币对变化时根据最新汇率重新计算目标金额。
2. 走势折线图切换时间范围时缺少动画过渡。TrendChart.jsx 中 7 天/30 天/90 天切换仅通过 setRangeIdx 触发 useMemo 重算 trendData 和 layout，SVG path 直接突变，无 CSS transition 或 opacity 渐变效果。提示词第 4 条明确要求"切换时间范围时图表动画过渡"，当前完全忽略了这一需求。正确做法是给 SVG path 的 d 属性加 transition，或在数据切换时加 opacity 渐变动画。
3. searchCurrencies 函数中对 nameEnLower 变量重复调用 toLowerCase。在 currencyUtils.js 中第 45 行已将 nameEnLower 设为 c.nameEn.toLowerCase()，但第 54、55、61 行又写了 nameEnLower.toLowerCase()，对已小写的字符串再次执行 toLowerCase 虽是空操作不会导致功能错误，但代码冗余且反映编写时不够仔细。同时 formatDateTick 函数中 if 和 else 两个分支返回的格式字符串完全相同，else 分支多余。
