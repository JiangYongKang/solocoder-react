产物不满意：
1. TrendChart.jsx 的 interpolateLayout 在动画过渡时产生 NaN 坐标导致折线消失。resampleData 只从 fromLayout.points 拷贝 date 和 value，丢弃了 y 坐标，返回的 fromPoints 只有 {date, value, x} 缺少 y 属性。interpolateLayout 计算中间帧时使用 fromPt.y + (to.y - fromPt.y) * p，fromPt.y 为 undefined 得 NaN，传入 SVG path 后折线渲染失败，500ms 动画期间图表空白。需求要求切换时间范围时图表动画过渡，因 NaN 无法展示有效动画。正确做法是在 resampleData 中同步插值 y 坐标，即 result.push 时计算 curr.y + (next.y - curr.y) * frac。

2. index.jsx 的 handleSwap 互换金额时仅将旧值直接赋值，未调用 convertCurrency 做真正的重新换算。提示词要求"将原来的目标金额 7.2456 作为基准金额重新换算"，"重新换算"强调需要基于新基准金额和新货币对重新计算目标值。如果用户之前手动修改过目标金额，直接交换旧值会导致显示不正确。正确做法是在设置基准金额后调用 convertCurrency 计算新的目标金额。

3. currencyUtils.js 的 formatDateTick 函数中存在无效果的 if 分支死代码。函数接收 totalDays 参数并判断 totalDays <= 7，但两个分支返回完全相同的格式字符串 M/D，if 判断没有产生任何实际差异，equals 分支毫无意义。虽然这个函数并非本轮修复的范围，但模型在编辑同文件的 searchCurrencies 函数时有机会注意到这个冗余分支却忽略未处理，遗漏了清理死代码的机会。

过程不满意：
1. 模型因 lint 错误对 TrendChart.jsx 进行了六次完整重写而非最小化修正。每次遇到一个 lint 错就全量重写——第一次未使用变量和 ref 问题、第二次 useEffect 中 setState 错误、第三次跨任务查看无动画的 TemperatureChart.jsx 后继续重写、第四次 class 拼接 bug。这种试错式调试没有一次性列出所有错误后再集中修复。正确做法是先收集全部 lint 错误清单，理解每个错误含义，制定针对性的最小化修复方案。

2. 模型跨任务查看了 TemperatureChart.jsx（src/pages/weather/ 目录下的组件）试图寻找动画实现参考，但 TemperatureChart 组件中没有任何动画逻辑，查看这个组件完全是无效操作，浪费了时间和 token。不同任务之间有严格的包隔离要求，模型不应在解决当前任务的问题时去参考其他任务的组件实现。

3. 模型在首次运行测试时执行了 pnpm run test --run 命令，因为 package.json 中 test 脚本已定义为 vitest --run，再次追加 --run 导致命令出错，需要重新执行正确的 pnpm run test 命令。模型没有先检查 test 脚本的实际定义就盲目追加参数，属于测试执行环节的粗心操作。正确做法是在运行测试命令前先确认 scripts 中的实际定义，确保参数组合正确。
