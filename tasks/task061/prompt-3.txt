你实现的实时汇率换算器经过上一轮修复后，还存在以下问题需要进行修复：

1. 走势折线图切换时间范围时图表消失。从 7 天切换到 30 天或 90 天时，resampleData 函数从旧布局数据点中上采样生成中间帧点位时，只拷贝了 date 和 value 属性，没有把原始数据点中的 y 坐标一起带入新点位。导致 interpolateLayout 在计算动画中间帧的 y 坐标时使用了 undefined 值，产生 NaN 传入 SVG path 的 d 属性，整个折线在 500ms 动画期间完全不可见。需要确保上采样后的数据点保留完整的坐标信息供动画插值使用。
2. 互换按钮点击后仅将旧的 targetAmount 和 baseAmount 做了交换，没有根据新的货币对和新的基准金额重新调用换算逻辑计算目标金额。当用户手动修改过目标金额使其不同于自动推导值时，交换旧值会导致显示错误。
3. 修复后重新运行 pnpm run test，需要让所有测试通过。
