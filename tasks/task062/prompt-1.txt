在当前项目中实现习惯养成追踪，该页面是首页任务卡片"习惯养成追踪"点击后的二级页面，使用 localStorage 持久化习惯数据和打卡记录。

这个页面是一个个人习惯养成追踪工具，用户可以创建习惯、每日打卡，并以热力图和统计图表方式可视化追踪进度。

1. 习惯创建：页面顶部有一个「新建习惯」按钮，点击后弹出表单——包含习惯名称输入框、习惯描述（选填）、习惯图标选择（提供一组 emoji 如 🏃📚💧🧘🎸✍️💤🚿🥗）、目标频率选择（每天/每周 X 次/每月 X 天）。创建后的习惯以卡片形式展示在主区域，卡片上显示习惯名称、图标、目标频率、当前连续打卡天数。
2. 打卡日历热力图：每个习惯卡片下方展示该习惯的打卡热力图，样式模仿 GitHub 贡献图——横向为周排列、纵向为星期几，每个小方格代表一天，打卡过的小方格填充颜色（颜色深浅代表当日打卡次数或完成度），未打卡的为浅灰色。热力图默认展示最近 365 天的打卡数据，可通过左右箭头切换年份查看。鼠标悬停在方块上显示日期和打卡详情。当天的小方格有特殊边框标记。
3. 完成率统计进度条：每个习惯卡片上展示本周和本月的完成率进度条。进度条根据目标频率计算——例如每天目标表示本周目标 7 次、本月目标 30 次。已完成次数 / 目标次数 = 完成率。进度条颜色根据完成率变化——0-60% 灰色、60-80% 蓝色、80-100% 绿色、超过 100% 金色。进度条旁显示具体数字如「本周 5/7」。页面顶部还有一个总览区，以环形图汇总展示所有习惯的整体完成率。
4. 连胜记录：每个习惯卡片上突出展示当前的连续打卡天数（Streak）。连续天数以火焰图标 🔥 配合数字展示。如果昨天未打卡，连续天数重置为 0。同时显示历史最长连续天数记录，用于激励用户打破记录。达到 7 天、30 天、100 天里程碑时有特殊的徽章标记展示在卡片上。
5. 打卡提醒模拟：每个习惯卡片上有一个闹钟图标按钮，点击后弹出提醒时间设置——选择每天几点提醒（如 8:00、12:00、20:00）。开启提醒后，页面使用定时器模拟提醒——到达设定时间后，页面顶部显示通知横幅「该打卡啦！—— [习惯名称]」，5 秒后自动消失。支持同时为多个习惯设置不同提醒时间。提醒开关和数据持久化到 localStorage。
6. 归档激活切换：习惯卡片上有「归档」按钮，点击后该习惯不再出现在活跃列表中，移到「已归档」标签页中。已归档的习惯保留历史打卡数据，但不参与活跃统计。在归档标签页中，可以点击「激活」将习惯恢复到活跃列表。归档前的打卡记录不变，连续天数会在重新激活后根据最近打卡记录重新计算。页面顶部有「活跃」和「已归档」两个标签页切换。
7. 代码实现需要放在 src/pages/habit-tracker/ 下，测试代码放在 src/pages/__tests__/habit-tracker/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。
