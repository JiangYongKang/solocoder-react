在当前项目中实现课程表排课，该页面是首页任务卡片"课程表排课"点击后的二级页面，使用 localStorage 持久化课程安排数据。

这个页面是一个高校或培训机构的课程表制作工具，用户可以按周视图排课、拖拽课程到时间格、检测教室冲突，并支持导出打印。

1. 周视图网格：页面主体为一张周视图课表——横轴为星期列（周一至周日，共 7 列），纵轴为时间段行（如 08:00-08:45、08:50-09:35 直至 20:00，每个时间段为 45 分钟，含课间休息 5 分钟，共约 16-18 个时间段）。网格左上角显示「时间」和「星期」角标。每条网格线清晰可见，当前时间对应的行高亮浅色背景。周末列（周六周日）有浅灰色背景以示区分。课表区域支持垂直滚动和水平滚动（如果屏幕宽度不够），表头（星期行）在垂直滚动时固定可见。
2. 课程拖拽添加：课表左侧有未安排的课程列表（课程池），每个课程是一个可拖拽的卡片——显示课程名称、教师、教室和时长（以时间段数量表示，如 2 节）。将课程卡片拖拽到课表网格中的某个单元格时，该课程占据从此单元格开始的连续 N 个时间段（N 为课程时长），单元格合并显示课程信息。课程卡片拖拽过程中目标区域高亮显示占位。课程在课表中的布局用不同颜色区分——不同课程自动分配不同颜色，颜色在课程创建时确定。
3. 颜色标记与详情弹窗：每个课程在添加时可选择颜色标记（预设 10 种学科颜色如蓝色数学、红色语文、绿色英语等）。已排入课表的课程卡片上显示课程名称、教师、教室、时间。点击课表中的课程卡片弹出详情弹窗——显示课程完整信息（名称、教师、教室、时间段、学分、备注）。详情弹窗中提供「编辑」和「从课表移除」按钮。编辑模式下可以修改课程的周次（如单周/双周/全周）、时间段调整。从课表移除的课程回到左侧课程池。
4. 教室冲突检测：排课时系统自动检测教室冲突——同一时间段同一教室不能安排两门课程。如果拖拽课程到已占用的教室和时间段，目标单元格显示红色闪烁提示，课程无法放置并弹回原位，同时显示提示「教室 [教室名] 在 [时间段] 已被 [课程名] 占用」。同理检测教师冲突——同一时间段同一教师不能安排两门课程。冲突检测信息展示在课表下方的一个警告面板中——列出所有检测到的冲突（教室冲突和教师冲突），每条冲突显示具体时间、教室或教师、两门冲突的课程名称。
5. JSON 导入导出：页面顶部工具栏中有「导出 JSON」和「导入 JSON」按钮。导出 JSON 将当前完整的课程安排（课程列表、排课信息、颜色配置、教室列表、教师列表）导出为一个 JSON 文件下载。导入 JSON 可以从本地 JSON 文件恢复完整课表状态。导入时校验 JSON 格式和数据结构完整性——如果数据不完整或格式错误，显示具体错误原因。导入成功后重新进行冲突检测。
6. 打印友好视图：页面提供「打印视图」按钮，点击后切换到打印友好的课表视图——隐藏工具栏、课程池、警告面板等编辑 UI，仅保留课表网格。课表在打印视图中自动适配 A4 纸张比例，字体大小适配打印。打印视图提供浏览器原生的 window.print() 调用，用户可在打印对话框中选择另存为 PDF。打印视图中背景色简化、文字用黑色、网格线保留。打印视图中有「返回编辑」按钮退回到正常编辑视图。
7. 代码实现需要放在 src/pages/schedule-planner/ 下，测试代码放在 src/pages/__tests__/schedule-planner/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。
