在当前项目中实现 WebSocket 调试工具，该页面是首页任务卡片"WebSocket 调试工具"点击后的二级页面，使用 localStorage 保存连接历史。

这个页面是一个 WebSocket 协议的联调和测试工具，用户可以通过浏览器直接建立 WebSocket 连接、发送和接收消息、监控连接状态，全程在前端完成。

1. 连接 URL 与状态指示：页面左侧或顶部为连接管理区域——包含一个 WebSocket URL 输入框（默认值为 ws://echo.websocket.org，这是一个真实存在的公开测试端点，请确保使用 wss:// 协议连接到真实的公开 echo 测试服务器，或使用模拟连接进行演示）、「连接」按钮、「断开」按钮。连接状态以彩色指示灯显示——绿色圆点 +「已连接」表示连接成功、黄色圆点 +「连接中...」表示正在建立连接、红色圆点 +「已断开」表示未连接或断开、灰色圆点 +「错误」表示连接失败。状态指示灯旁显示连接持续时长（已连接状态的累计时间）。连接失败时显示失败原因（如「服务器无响应」「握手失败」）。
2. JSON 消息发送：连接成功后，下方消息发送区域激活（连接前呈灰色禁用状态）。发送区域包含——消息内容输入框（多行文本，支持 JSON 格式）、一个「格式化 JSON」按钮（点击后自动格式化输入框中的 JSON 文本并缩进显示）、「发送」按钮。点击发送后，如果输入框内容为有效 JSON，消息以 JSON 格式发送；如果内容不是有效 JSON，以纯文本发送。发送的消息下方有一个消息模板快捷面板——提供几个常用 JSON 消息模板（如 {"type": "ping"}、{"type": "subscribe", "channel": ""}），点击模板自动填入输入框。发送按钮可以使用快捷键 Ctrl+Enter。
3. 实时滚动日志：页面主体为消息日志区域——以滚动列表形式展示所有发送和接收的消息。每条日志记录包含——时间戳（精确到毫秒的时分秒格式如 14:32:05.123）、方向标记（发送标记为 ↑ 箭头并着蓝色、接收标记为 ↓ 箭头并着绿色）、消息内容（JSON 格式化缩进显示或纯文本）。消息内容较长时默认折叠显示摘要（前 100 个字符），点击展开显示完整内容。新消息到达时自动滚动到日志底部。如果用户手动滚动到历史位置查看旧消息，新消息到达时不强制滚动到底部，但在日志底部显示「↓ 有新消息」的浮动按钮，点击回到最新位置。日志区域顶部有搜索框——输入关键词过滤日志，匹配的行高亮显示。
4. 连接历史：页面侧边栏或下拉列表中展示连接历史——自动记录所有连接过的 URL（不包含消息内容）。每条历史记录包含 URL、最后连接时间。点击历史项可快速填入 URL 输入框。支持删除单条历史记录和清空全部历史。当前连接的 URL 在历史列表中高亮。连接历史持久化到 localStorage。
5. 心跳检测：连接成功后自动启动心跳检测——每隔 10 秒发送 ping 消息（如果连接的是真实 echo 服务器则不适用心跳，此处针对模拟连接场景）。心跳检测的状态在连接状态区域显示——最近一次心跳的时间和结果（成功/超时）。连续 3 次心跳超时后，连接状态变为红色，显示「连接已断开（心跳超时）」并自动断开连接。心跳间隔和超时时间可配置——在设置面板中可调整心跳间隔（5-60 秒）和超时阈值（连续失败次数，1-10 次）。
6. 自动重连配置：在连接断开时（非用户主动断开），如果开启了自动重连，系统自动尝试重连。重连配置在设置面板中——重连开关、最大重连次数（1-20 次）、重连间隔（1-30 秒）。重连过程中状态灯闪烁黄色，消息日志中插入系统消息「[系统] 正在重连（第 X 次）...」。重连成功后日志中插入「[系统] 重连成功」。达到最大重连次数仍未成功后，停止重连并显示「[系统] 重连失败，已达最大重连次数」。
7. 代码实现需要放在 src/pages/websocket-debugger/ 下，测试代码放在 src/pages/__tests__/websocket-debugger/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。
