你实现的 WebSocket 调试工具还存在一些问题，需要进行修复：

1. 连接意外断开时，ws.onclose 回调设置了 DISCONNECTED 状态而非 ERROR 状态，但错误原因的显示条件只检查 ERROR 状态，导致连接失败的原因信息在 UI 上永远看不到。同时 ws.onerror 设置的 ERROR 状态在随后的 onclose 中也被覆盖了。
2. 搜索框只能过滤消息列表，但匹配的关键词没有在消息内容中高亮显示。CSS 中已定义了高亮样式类但未被 JSX 引用。需求要求"匹配的行高亮显示"，目前只有过滤没有高亮。
3. 建立连接后在 ws.onopen 中无条件启动了心跳检测。需求明确说明对于真实 echo 服务器不适用心跳，只有模拟连接场景才使用。当前连接 echo.websocket.org 等真实服务器时也会发送 ping 消息，与需求不符。
4. 心跳超时检测中，ws.onmessage 收到任意消息都会清空失败计数，不区分是否为 pong 回包。服务器推送的普通消息也会错误地重置心跳超时计数，导致心跳超时检测失效。
5. 修复后重新运行 pnpm run test，需要让所有测试通过。
