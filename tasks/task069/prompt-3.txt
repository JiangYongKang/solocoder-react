你实现的 WebSocket 调试工具经过上一轮修复后，还存在以下问题需要进行修复：

1. highlightKeyword 函数使用了递归调用来高亮文本中所有匹配的关键词位置。当消息内容较长时（如几千行 JSON），匹配次数越多递归深度越深，有栈溢出风险。应避免在文本处理中使用递归。
2. ws.onclose 意外断开时设置的 errorReason 只使用了固定中文文案"连接意外断开"，没有利用 WebSocket CloseEvent 对象中标准的 code 和 reason 字段。开发者无法从错误信息中区分服务端主动关闭（1000）、协议错误（1002）、还是网络中断（1006），调试信息不足。错误提示需要同时包含 code 码和 reason 原文。
3. 修复后重新运行 pnpm run test，需要让所有测试通过。
