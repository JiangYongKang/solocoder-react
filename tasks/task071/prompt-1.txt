在当前项目中实现 Markdown 笔记应用，使用 localStorage 持久化笔记数据。

这个页面是一个完整的笔记应用，用户可以创建多本笔记本并以树形目录组织笔记，支持 Markdown 编辑与实时预览、标签分类、全文搜索，以及笔记间的内部链接跳转。

1. 多笔记本树形目录：页面左侧显示树形目录结构——顶层为笔记本，每个笔记本下可以有文件夹和笔记的任意嵌套层级。支持新建笔记本、文件夹和笔记，右键菜单支持重命名和删除。节点支持拖拽移动（将笔记或文件夹拖到另一个笔记本或文件夹下）。点击笔记节点在主区域打开编辑。文件夹和笔记本节点支持展开折叠，展开状态持久化到 localStorage。
2. 编辑器与预览分栏：选中笔记后主区域分为左右两栏——左侧为 Markdown 源码编辑器（使用 textarea），右侧为 Markdown 渲染预览（支持标题、加粗、斜体、列表、代码块、表格、图片、链接等常见语法）。编辑器内容变更时预览实时更新。编辑内容自动保存到 localStorage（500ms 防抖）。分栏宽度支持拖拽调节，分栏比例持久化到 localStorage。
3. 标签分类：每篇笔记支持添加多个标签。标签在编辑器顶部以标签栏展示，支持输入新标签添加和点击 x 删除。左侧目录面板底部提供标签列表（展示所有使用过的标签及对应笔记数量），点击标签筛选显示该标签下的所有笔记，支持多标签组合筛选。
4. 全文搜索高亮：页面顶部提供搜索框，输入关键词后实时搜索所有笔记的标题和正文。搜索结果以列表展示——每条结果显示笔记标题、所在笔记本路径、匹配片段（关键词高亮）。点击结果项跳转到对应笔记并打开编辑器。搜索结果空时显示空状态提示。搜索索引在加载笔记时构建缓存，避免每次搜索都遍历全文。
5. .md 文件导入导出：工具栏提供导入按钮——点击后打开文件选择器，选择 .md 文件后内容读取到当前编辑器中作为新笔记，文件名作为笔记标题。导出按钮——将当前笔记导出为 .md 文件下载，文件名使用笔记标题。同时支持批量导出——选中笔记本后可将该笔记本下所有笔记导出为一个 ZIP 包（此处可用单个文件模拟，实际前端通常用 JSZip）。
6. 笔记间链接跳转：Markdown 渲染器支持内部链接语法 [[笔记标题]]——渲染为可点击的链接样式，点击后跳转到对应笔记并打开编辑器。如果链接指向不存在的笔记，渲染为红色虚线链接样式，点击后自动创建一篇以该标题为名的新笔记。笔记重命名或删除后，其他笔记中指向它的链接需要处理为失效状态并视觉提示。
7. 代码实现需要放在 src/pages/markdown-notes/ 下，测试代码放在 src/pages/__tests__/markdown-notes/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。
