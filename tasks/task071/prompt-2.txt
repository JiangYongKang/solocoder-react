你实现的 Markdown 笔记应用存在以下问题，需要进行修复：

1. 搜索结果存在 XSS 安全漏洞：SearchBar.jsx 中搜索结果的关键词高亮使用了 dangerouslySetInnerHTML，直接插入了未经过 HTML 实体转义的笔记标题和内容片段。当笔记标题包含 HTML 标签或脚本标签时，搜索结果会执行注入的代码。修复时需要对匹配内容先进行转义再包裹高亮标签。

2. Markdown 渲染存在嵌套错误：noteUtils.js 中的渲染函数在代码块（``` 包围区域）内部仍然对反引号执行了行内代码高亮匹配，导致代码块内的 `<code>` 标签被额外嵌套一层，破坏了 HTML 结构。修复时需要确保代码块范围内的内容不被行内代码正则二次匹配。

3. 保存逻辑存在竞态覆盖：useEffect 在切换笔记时立即全量保存，同时编辑器还维护了 500ms 防抖保存回调。当用户快速切换笔记后，前一笔记的防抖回调可能晚于切换笔记后的即时保存执行，导致新笔记内容被旧笔记的快照覆盖。修复时需要确保保存操作不会用过期数据覆盖当前数据。

4. 修复后需要为该页面补充纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。测试代码放在 src/pages/__tests__/markdown-notes/ 下。

5. 确保所有子页面在 375px 移动端宽度下布局不崩坏，使用 pnpm run dev 启动后手动验证关键页面。

6. 重新运行 pnpm run test，需要让所有测试通过。
