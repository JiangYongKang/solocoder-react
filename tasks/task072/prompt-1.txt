在当前项目中实现 JSON 转 TypeScript 类型生成器，使用 localStorage 持久化历史记录。

这个页面是一个开发者工具，用户粘贴或编辑 JSON 数据后自动生成对应的 TypeScript 接口（interface）或类型别名（type），支持递归类型推导、字段必填/可选识别、类型命名自定义，并提供一键复制和历史记录功能。

1. JSON 输入编辑器：页面左侧提供 JSON 编辑区——一个带有行号的代码编辑器样式的 textarea，支持粘贴 JSON 文本或手动输入。编辑器下方有「格式化」按钮，点击后对编辑器中的 JSON 进行格式化（缩进美化）。如果 JSON 格式不合法，在编辑器下方显示具体的解析错误信息（包含错误位置行号），不生成的类型。
2. 类型推导与输出：右侧为 TypeScript 类型输出区，内容随左侧 JSON 变更实时更新（300ms 防抖）。需要正确处理 JSON 的各种结构——对象转为 interface，数组转为 type (type Foo = ItemType[])，嵌套对象递归生成嵌套 interface/type，深度超过 3 层时可使用 type 别名引用。JSON 中的 null 值字段类型应推导为 T | null。数字类型区分整数（number）和浮点数（number），不区分具体数值。空数组 [] 推导为 any[]。空对象 {} 推导为 Record<string, any>。
3. 可选必填字段识别：如果 JSON 数据是数组且包含多个对象元素，工具应对比所有元素中字段的差异——所有元素都有的字段标记为必填，只在部分元素中出现的字段标记为可选（属性名后加 ?）。如果只有一个对象元素，所有字段默认为必填。支持在 JSON 编辑器中通过 JSON Schema 风格的注释标记来覆盖自动判定——例如在字段行上方添加 // @optional 强制标记为可选。
4. 命名自定义：生成的类型名称需要可自定义——顶部提供一个「根类型名称」输入框，默认为 RootType。如果 JSON 是数组，容器的类型名可以单独设置，默认为 RootTypeList。子对象类型默认使用父类型名 + 字段名（PascalCase 格式），也可以点击生成的类型名直接编辑修改。如果手动修改了某个内嵌类型名，在 JSON 结构不变的情况下后续不会覆盖手动修改的名称。
5. 一键复制与操作：输出区域上方提供「复制」按钮——点击后复制生成的 TypeScript 代码到剪贴板，并显示「已复制」提示 2 秒后消失。提供「下载 .ts 文件」按钮——下载为 .ts 文件。提供「只生成 interface」和「优先使用 type」的切换开关，控制生成输出类型偏好。
6. 历史记录：页面底部提供历史记录面板——每次生成（JSON 内容变更稳定后自动记录）保存一条记录，记录包含时间戳、JSON 片段摘要（前 80 字符）、根类型名称。历史记录以列表展示，支持点击回退到某条历史记录（恢复当时的 JSON 和生成的类型）。支持单条删除和全部清空。最多保存最近 30 条记录。历史数据持久化到 localStorage。
7. 代码实现需要放在 src/pages/json-to-ts/ 下，测试代码放在 src/pages/__tests__/json-to-ts/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。
