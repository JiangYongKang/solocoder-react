你实现的 JSON 转 TypeScript 类型生成器存在以下问题，需要进行修复：

1. 递归收集嵌套类型时缺少深度保护：collectNestedObjects 函数在处理深层嵌套的 JSON（如超过 200 层嵌套对象）时没有深度上限检查，会持续递归直到调用栈溢出导致页面崩溃。修复时需要添加递归深度硬限制。

2. 根数组类型的名称设置存在缺陷：当根 JSON 是包含数组字段的对象（如 {"items": [1,2,3]}）而非直接数组时，generateTypeScript 中 rootListName 无法正确赋值，导致生成的代码中容器类型别名缺失。修复时需要确保在嵌套对象中找到顶级数组类型时也能正确设置列表容器名称。

3. copyToClipboard 和 downloadTsFile 两个工具函数完全没有单元测试。这两个函数都包含多分支逻辑（浏览器/SSR 环境判断、API 回退等），缺少测试意味着各环境的正确性未验证。修复后需要为这些函数补充测试。测试代码放在 src/pages/__tests__/json-to-ts/ 下。

4. 确保所有子页面在 375px 移动端宽度下布局不崩坏，使用 pnpm run dev 启动后手动验证关键页面。

5. 重新运行 pnpm run test，需要让所有测试通过。
