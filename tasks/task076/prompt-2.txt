你实现的版本发布管理系统存在以下问题，需要进行修复：

1. 审批操作存在快速双击绕过的风险：useReleaseManager.js 中 handleApprove、handleRollback 等回调函数通过 useCallback 创建闭包，操作校验使用闭包中捕获的 releases 数组。当用户快速双击"审核通过"等按钮时，两次点击使用的都是闭包中同一个旧的 releases 快照做状态校验，可能绕过审批权限检查导致重复审批。修复时需要确保每次操作的校验都基于最新状态。

2. Diff 对比面板缺少大文本保护：DiffPanel.jsx 中的行级 Diff 使用了 LCS（最长公共子序列）动态规划算法。当两个版本的变更日志超过数百行 Markdown 时，LCS 矩阵计算耗时急剧增加，会导致页面明显卡顿。修复时需要为 Diff 输入添加长度保护，或对超长文本做截断提示。

3. 所有审批操作的操作人都硬编码为同一用户：版本提交、审核、回滚等不同角色本应由不同用户执行，但当前所有状态变更记录中的操作人都标记为同一个常量。修复时需要提供用户身份的输入或切换方式。

4. 整个版本发布管理页面及其所有子组件在 375px 移动端宽度下布局崩坏——表格列表横向溢出、Diff 左右分栏挤压、审批弹窗过宽。修复时需要为移动端补充响应式样式。

5. 修复后需要为该页面补充纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。测试代码放在 src/pages/__tests__/release-manager/ 下。

6. 重新运行 pnpm run test，需要让所有测试通过。
