你实现的 RPG 角色创建器存在以下问题，需要进行修复：

1. 属性衍生数值的返回类型不一致：utils.js 的 calculateDerivedStats 函数中，hp、physicalAttack、magicalAttack 等字段返回数字类型，而 critRate 和 dodgeRate 返回字符串类型（如 "2.5%"）。同一函数的不同字段返回不同类型，调用方在使用这些衍生数值做进一步计算时会产生 NaN。修复时需要统一返回数字类型，格式化逻辑交由展示层处理。

2. 角色卡片导出函数在测试环境中不可执行：drawCardCanvas 函数内部直接调用了 document.createElement('canvas')，这使得该函数依赖浏览器 DOM API，在 Node.js 测试环境中完全无法运行和测试。修复时需要将卡片数据计算逻辑与 Canvas 绘制逻辑分离，或将 DOM 操作抽象为可注入的依赖。

3. 整个角色创建器页面及其所有子组件在 375px 移动端宽度下布局崩坏——属性面板的加减按钮挤在一起无法点击、技能树节点连线溢出、外观选项横向滚动不足。修复时需要为移动端补充响应式样式。

4. 修复后需要为该页面补充纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。测试代码放在 src/pages/__tests__/rpg-creator/ 下。

5. 重新运行 pnpm run test，需要让所有测试通过。
