你实现的数据脱敏工具存在以下问题，需要进行修复：

1. 多条规则按顺序替换时会产生相互干扰：dataMaskUtils.js 的 applyRules 函数按规则顺序逐个对文本做正则替换。当先执行的规则将敏感信息替换为星号后，后续规则的正则可能匹配到这些星号字符串而产生误替换。修复时需要确保各条规则的匹配和替换不会相互干扰。

2. buildHighlightSegments 函数接收了 maskedText 参数但完全没有使用它：该函数内部所有高亮位置计算都基于 originalText，maskedText 作为一个死参数会误导调用方。修复时需要移除这个无用参数，或在函数内部使用它修正高亮区间。

3. 自定义规则中使用捕获组替换模板（$1 $2 等）的功能实际无效：用户在添加自定义规则时可以输入正则表达式和替换模板，但 applyRule 函数在做替换时使用的 replacement 被硬编码为固定字符而非用户输入的捕获组模板，导致自定义规则的替换逻辑完全不按用户配置执行。修复时需要使自定义规则的替换模板参数实际生效。

4. 修复后需要为该页面补充纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。测试代码放在 src/pages/__tests__/data-mask/ 下。

5. 确保所有子页面在 375px 移动端宽度下布局不崩坏，使用 pnpm run dev 启动后手动验证关键页面。

6. 重新运行 pnpm run test，需要让所有测试通过。
