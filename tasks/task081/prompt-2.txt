你实现的视频会议模拟页面还存在一些问题，需要进行修复：

1. videoMeetingCore.js 中导出的 drawParticipantCanvas 函数完全没有被测试覆盖，测试文件中既没有 import 也没有为该函数编写任何用例。该函数是 Canvas 画面渲染的核心入口，内部包含多个绘制路径分支。需要为该函数补充单元测试，至少覆盖 Canvas context 为 null 时的防御行为以及不同参与者状态（正常、视频关闭、静音、举手）下是否正确调用了对应的绘制路径。

2. 聊天消息的自动模拟定时器只在聊天面板打开时才调度，关闭侧边栏后定时器被清除，不再有消息产生。提示词要求的是定时器每 5-15 秒随机生成一条模拟消息，不论面板是否打开都应该持续模拟。同时面板关闭时无法收到新消息导致未读角标永远不会增长，形成功能死码。

3. 底部控制栏的三个 toggle 操作中，静音和举手都调用了 videoMeetingCore.js 的 toggleParticipantProperty 纯函数，但屏幕共享的 toggle 在 index.jsx 中直接内联了 setState 逻辑，没有使用同一个纯函数。应统一通过 toggleParticipantProperty 处理屏幕共享的状态切换。

4. 侧边栏的聊天按钮和参会人按钮只能打开面板不能关闭，点击第二次没有反应。用户只能通过面板内部的关闭按钮来关闭面板。应改为开关切换行为——点击已打开的面板按钮时关闭面板。

5. 修复后重新运行 pnpm run test，需要让所有测试通过。
