在当前项目中实现代码执行沙箱模拟页面，使用 localStorage 持久化代码和执行历史。Python 代码的实际执行在前端用 JavaScript 模拟。

这个页面模拟一个在线代码运行环境，用户可以编写和运行 JavaScript 或 Python 代码（Python 为前端模拟执行）、查看输出和耗时，以及管理代码片段的保存加载和执行历史。

1. 语法高亮编辑器：页面主体左侧为代码编辑器——使用 textarea 实现，但需要通过叠加层或 CSS 模拟语法高亮效果。支持 JavaScript 和 Python 两种语言的基础语法高亮——关键字（JS 的 function/const/let/var/if/else/for/while/return、Python 的 def/class/if/elif/else/for/while/return/import）、字符串（单引号和双引号内的内容）、注释（JS 的 // 和 /* */、Python 的 #）、数字字面量。高亮颜色方案——关键字蓝色、字符串绿色、注释灰色、数字橙色。编辑器支持 Tab 缩进（插入 2 个空格而非 Tab 字符）和括号自动补全（输入 (、[、{ 时自动补全右括号）。
2. JS/Python 语言选择：编辑器上方提供语言选择下拉框——JavaScript 和 Python 两个选项。选择 Python 时执行逻辑为前端模拟——通过 JavaScript 解析并模拟 Python 常用函数的输出（如 print 转换为 console.log 输出、range/len/sum/max/min 等常用内置函数用 JS 等价实现模拟，无法模拟时返回"该函数暂不支持"）。选择 JavaScript 时直接使用 eval 或 new Function 执行用户代码（需用 try-catch 包裹捕获执行错误）。
3. 执行与输出控制台：编辑器右侧为输出控制台——显示代码执行的标准输出和错误信息。正常输出以白色/默认文本颜色显示，错误信息以红色文本显示。控制台支持清空按钮。输出区域自动滚动到底部显示最新输出。多行输出逐行显示，以 > 符号为行前缀。
4. 耗时显示：每次执行后输出控制台底部显示执行耗时——使用 performance.now 或 Date.now 测量代码执行前后的时间差，以毫秒为单位显示。Python 模拟执行也需显示耗时。
5. 保存加载：工具栏提供「保存」按钮——将当前编辑器中的代码保存到 localStorage，弹窗输入代码片段名称。提供「加载」按钮——从已保存的代码片段列表中选择加载到编辑器。代码片段列表显示名称、语言、保存时间和代码行数。支持重命名和删除已保存的片段。最多保存 50 个片段。
6. 执行历史：输出控制台下方提供执行历史面板——按时间倒序展示最近 50 次执行记录。每条记录显示执行时间、语言、代码前 50 个字符（截断）、执行耗时、成功/失败状态。点击历史记录可加载该次执行的代码到编辑器中。历史数据持久化到 localStorage。
7. stdin 输入收缩：输出控制台上方提供一个可折叠的 stdin 输入区域——点击展开后显示 textarea 用于输入标准输入内容，如果代码中使用了模拟的 input/prompt 读取输入则从此区域读取。折叠时仅显示一行提示"标准输入（已折叠，点击展开）"。
8. 代码实现需要放在 src/pages/code-sandbox/ 下，测试代码放在 src/pages/__tests__/code-sandbox/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。
