你实现的排序算法可视化页面还存在一些问题，需要进行修复：

1. index.jsx 的 scheduleNext 函数在对比模式下存在调度缺陷：B 侧的 setTimeout 回调中没有递归调用 scheduleNext，导致调度循环完全由 A 侧驱动。当 A 侧先完成排序而 B 侧仍在运行时，A 侧回调仅设置了 B 侧定时器，但 B 侧回调执行后不会继续调度自己，B 侧只能执行额外一步就停止，算法 B 在对比模式下无法独立完成排序。两侧的调度逻辑应保持对称。

2. index.jsx 的 scheduleNext 函数在对比模式下递归调用时直接覆盖了 timerBRef.current，但未清除旧的 B 侧定时器句柄。如果旧定时器仍处于 pending 状态，会导致 B 侧同时有两个定时器存在，B 侧因旧定时器额外触发而执行多余步骤，两个算法在视觉上执行步调不一致。

3. index.jsx 的 stepGeneratorOnce 函数长达 125 行，在一个函数内混杂了状态转换、generator 创建与初始化、完成状态处理、普通步骤更新、日志生成、算法名称查找等多种不同职责。应当按职责拆分为多个小函数以降低单函数复杂度。

4. 修复后重新运行 pnpm run test，需要让所有测试通过。同时需要启动 pnpm run dev 在浏览器中手动验证对比模式下两个算法是否同步执行到完成、暂停/继续的交互流程是否正常。
