在当前项目中实现团队 Wiki 页面，使用 localStorage 持久化所有数据。

这个页面是一个团队知识库管理工具，团队成员可以创建和管理空间、编写 Markdown 页面、浏览历史版本和进行全文搜索。

1. 空间创建与管理：页面左侧为空间列表侧边栏——显示所有 Wiki 空间（初始预置 2-3 个示例空间如"技术文档"和"产品需求"），每个空间显示名称和页面数量。支持创建新空间（输入名称和描述）、编辑空间信息和删除空间（删除前确认，关联的所有页面一并清除）。空间列表支持搜索过滤——输入关键词实时筛选空间名称。当前选中的空间高亮显示。
2. 页面树形层级与面包屑：空间内页面以树形结构展示——支持多级页面层级（父页面下可创建子页面，子页面可继续嵌套）。树形节点支持展开和折叠，当前浏览的页面节点高亮。页面区顶部显示面包屑导航——从空间根到当前页面的完整路径，每一级可点击跳转。
3. Markdown 编辑与渲染：页面内容区为左右分栏布局——左侧 Markdown 编辑器（textarea），右侧实时渲染预览。编辑器支持常用 Markdown 语法——标题、加粗、斜体、列表、链接、图片、代码块、表格。工具栏提供快捷插入按钮（加粗、标题、链接、代码块等）。编辑时自动保存到 localStorage（防抖 2 秒）。
4. 历史版本回溯与 Diff：每个页面自动记录历史版本——每次保存时创建新版本快照（存储页面标题和内容）。页面工具栏显示版本数量并提供"历史版本"按钮——打开版本列表侧边栏，显示各版本的时间和版本号。选择两个版本后以左右分栏 diff 对比展示差异——新增内容绿色高亮、删除内容红色高亮。支持从任意历史版本恢复到当前编辑区。
5. 全文搜索：顶部全局搜索栏——输入关键词后搜索所有空间中所有页面的标题和内容。搜索结果以列表展示——每条结果显示页面标题、所属空间名称、匹配内容片段（关键词高亮）。点击搜索结果跳转到对应页面并滚动到首次匹配位置。搜索结果按相关度排序——标题匹配权重高于内容匹配。
6. 标签分类：每个页面可添加多个标签（如"前端"、"API"、"入门指南"）。页面下方显示标签列表，点击标签可添加新标签或删除已有标签。空间侧边栏底部显示本空间所有标签的标签云——标签按使用频次显示不同大小。点击标签过滤只显示带有该标签的页面。
7. 成员权限展示：每个空间有成员列表——显示成员的名称、角色（管理员/编辑者/阅读者）和加入时间。空间设置面板中以表格展示所有成员及其权限。当前用户默认作为管理员。添加的成员仅为展示用（模拟数据），不涉及实际权限校验。
8. 代码实现需要放在 src/pages/wiki/ 下，测试代码放在 src/pages/__tests__/wiki/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑（如树形结构操作、版本管理、搜索匹配、标签操作等逻辑）。
