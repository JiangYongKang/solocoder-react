在当前项目中实现 2048 游戏，游戏状态在前端维护，使用 localStorage 持久化最高分记录。

这个页面是经典的 2048 数字合并游戏，玩家通过方向键滑动让相同数字的方块合并成更大数字，目标是达到 2048。

1. 4×4 网格：游戏区为一个 4×4 的网格，每个格子为一个固定大小的方形单元格。初始时随机在两个空格子中生成数字（90% 概率生成 2，10% 概率生成 4）。网格使用 CSS Grid 布局，每个格子有边框和浅色背景。已填充数字的格子根据数字大小显示不同背景色（如 2 为浅灰、4 为米色、8 为橙色，数字越大颜色越深，2048 为金色）。
2. 方向键滑动合并：支持方向键（上下左右）控制滑动方向。按下方向键后，所有非空格子沿滑动方向移动到底——相同数字的相邻格子合并为一个格子（值为两数之和），合并后的格子不再参与当次滑动的进一步合并。如果任意格子发生了移动或合并，在随机空格子中生成一个新数字。
3. 分数计算与随机生成：每次合并产生的数值累加到当前总分中（如两个 4 合并为 8，总分加 8）。当前得分实时显示在游戏区上方，同时显示历史最高分（从 localStorage 读取）。
4. 移动动画：方块移动和合并时使用 CSS transition 或 animation 实现平滑动画——格子从原位置滑动到新位置（持续时间约 150ms），合并时数字放大再缩小的弹跳效果。动画期间不允许新的方向键输入（防抖）。
5. 胜负判定：当任意格子的数字达到 2048 时判定为胜利——显示"恭喜达到 2048！"的胜利弹窗，询问玩家是"继续挑战"还是"重新开始"。当网格被填满且任意方向都无法移动时判定为失败——显示"游戏结束"弹窗，展示最终得分。
6. 撤销：提供"撤销"按钮——撤销上一步滑动操作，恢复到滑动前的网格状态和分数。最多支持撤销最近 5 步，超过 5 步则最早的一步被移除。撤销不计入最终得分（即撤销回退的操作不会影响分数历史）。
7. 最高分持久化：游戏结束时将当前得分与 localStorage 中的最高分比较，如果更高则更新最高分并显示"新纪录！"标记。游戏区始终显示历史最高分。
8. 代码实现需要放在 src/pages/game-2048/ 下，测试代码放在 src/pages/__tests__/game-2048/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑（如滑动合并算法、分数计算、胜负判定、撤销逻辑等核心游戏逻辑）。
