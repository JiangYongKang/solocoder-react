你实现的连连看游戏还存在一些问题，需要进行修复：

1. 消除后检测到死局时自动触发重排，但没有检查剩余重排次数也不更新已用次数——即使玩家用完了三次手动重排机会，消除后仍可触发自动重排。提示词要求"每次游戏最多 3 次重排机会"，自动重排也应受此限制。需在执行自动重排前检查 shufflesRemaining 并在重排后同步更新次数状态。
2. calculateScore 函数中提示奖励部分硬编码了 maxHints = 1，但重排奖励通过 MAX_SHUFFLES 常量准确计算。游戏中提示按钮无使用次数限制，这个硬编码值与实际游戏机制脱节，且与重排奖励的对称计算逻辑不一致。需修正提示奖励的计算方式。
3. calculateScore 函数中解构了 DIFFICULTY_CONFIG[difficulty] 存入 config 变量但后续计算中完全未使用该变量，存在无用赋值。
4. 修复后需要为该页面补充纯 JS 函数的单元测试，只测和页面渲染无关的逻辑。测试代码放在 src/pages/__tests__/link-game/ 下。
5. 重新运行 pnpm run test，需要让所有测试通过。
