在当前项目中实现 URL 解析编解码工具页面，所有状态仅在当前会话内维护，支持导出但不持久化。

这个页面是一个开发者常用的 URL 解析与编解码工具箱，用户可以拆解分析 URL 各部分、编辑查询参数并实时重建 URL、进行多种编码解码转换。

1. URL 各部分拆解展示：页面顶部为 URL 输入框——用户粘贴或输入一个 URL。下方自动拆解为各部分独立展示——协议（protocol，如 https:）、主机名（hostname，如 www.example.com）、端口号（port，如 8080，未指定时显示"默认"）、路径（pathname，如 /path/to/page）、查询字符串（search，如 ?name=value&id=1）、哈希（hash，如 #section）。每部分用独立卡片或字段展示，带标签说明。输入框内容变化时各部分实时更新拆解。
2. 查询参数键值对表格编辑：查询参数区域以可编辑表格展示——每行一个参数，包含键（Key）和值（Value）两列可编辑输入框。支持添加新参数行和删除已有参数行。参数修改后，URL 输入框中的完整 URL 实时重建更新。支持批量操作——一键清空所有参数。
3. 编码解码：URL 输入框下方的工具栏提供"URL 编码"和"URL 解码"按钮——对整个 URL 或选中的部分（如仅查询参数部分）进行 encodeURIComponent/decodeURIComponent 操作。编码和解码结果覆盖到 URL 输入框中。提供快捷按钮"仅编码参数"——只对查询字符串部分进行编码。
4. Base64 转换：提供 Base64 编解码区——一个独立文本输入框输入待编码文本，点击"Base64 编码"按钮输出编码结果，点击"Base64 解码"将 Base64 字符串还原为原文。编码结果和解码结果分别显示在输出框中，支持一键复制。编码失败或解码失败（非法 Base64 字符串）时显示错误提示。
5. JSON 互转：查询参数区域提供"参数 → JSON"按钮——将当前查询字符串的键值对转换为 JSON 对象格式（如 {"name":"value","id":"1"}）并显示在输出面板中。提供"JSON → 参数"按钮——粘贴 JSON 对象字符串，解析后转换为查询参数字符串更新到 URL 中。JSON 解析失败时显示错误提示，标识具体哪一行有语法问题。
6. 批量解析：页面底部提供批量解析面板——一个多行文本输入框，每行一个 URL。点击"批量解析"按钮后，以表格形式展示所有 URL 的解析结果——列包括序号、完整 URL、协议、主机名、路径、参数个数。支持导出为 CSV 文件下载。解析失败的行在状态列标红并显示失败原因。
7. 代码实现需要放在 src/pages/url-tool/ 下，测试代码放在 src/pages/__tests__/url-tool/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑（如 URL 解析、编码解码、Base64 转换、JSON 与查询参数互转等逻辑）。
