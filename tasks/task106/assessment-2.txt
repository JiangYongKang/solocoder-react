产物不满意：
1. svgPathEditorCore.js 中重写后的 deleteNode 函数使用了基于坐标值比较的定位方式，而辅助函数 getAnchorNodes 已经为每个锚点保存了 cmdIndex 字段。deleteNode 通过扫描 segments 并用 `i` 索引与锚点的坐标值 `x === seg?.x && y === seg?.y` 做比较来定位待删除的命令。当路径中不同锚点的坐标恰好相同时，坐标扫描会错误匹配到前面的命令而非用户选中的目标锚点。该问题的根本原因是 getAnchorNodes 已经提供了更精确的 cmdIndex 索引定位方式，但 deleteNode 却没有利用它，退回到了不稳定的坐标值匹配。正确做法是使用 `anchors[anchorIdx].cmdIndex` 代替坐标扫描来定位待删命令。
2. svgPathEditorCore.js 的 buildSvgString 函数在第 684 行使用了 `if (style.strokeWidth)` 来判断是否输出 stroke-width 属性，但 strokeWidth 值为 0 时是合法的 SVG 值（表示无描边），同时也是 JavaScript 中的 falsy 值。当用户在编辑器中设置描边宽度为 0 后导出 SVG，生成的代码中缺少 stroke-width 属性，浏览器会使用默认的 1px 渲染，导致导出的 SVG 在外部查看时与编辑器中的显示效果不一致（编辑器内无描边，导出的 SVG 却有 1px 描边）。正确做法是使用 `if (style.strokeWidth !== undefined && style.strokeWidth !== null)` 或 `if (typeof style.strokeWidth === 'number')` 来区别数字 0 和未设置的情况。
