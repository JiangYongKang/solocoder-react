产物不满意：
processAlerts 函数中持续时间计算的覆盖场景存在遗漏。index.jsx 的 processAlerts 函数有 if 和 else 两个分支，if 分支处理仍有告警触发的情况，else 分支处理全部告警解除的情况。模型新增的 resolveAlertRecords 调用只添加在了 else 全解除分支中，而 if 分支内虽然通过 inactiveRuleIds 变量正确识别了哪些规则已不再触发告警，但仅做了 delete 清理引用而没有同步调用 resolveAlertRecords 来更新对应记录的 duration 字段。当多个告警同时触发但只有部分解除时，已解除告警的持续时间永远不会被计算，duration 保持为 null，表格中持续时间列始终显示"-"，用户无法得知告警实际持续了多久。根本原因是模型在添加 resolveAlertRecords 调用时只关注了显式的全解除路径，没有对比检查 if 分支中的部分解除路径也需要同样处理。正确做法是在 if 分支的 inactiveRuleIds 清理逻辑后同步调用 resolveAlertRecords 计算持续时间并写回记录。
