你实现的多人协作文档编辑模拟页面还存在一些问题，需要进行修复：

1. 修订痕迹的视觉标记完全没有渲染到文档编辑区的文字上。CSS 中已定义了新增（绿色背景加下划线）、删除（红色背景加删除线）、格式修改（蓝色虚线边框）的样式，但 JSX 渲染逻辑中从未将这些样式类应用到文本上，也没有渲染悬停提示显示修改人和时间。需要将修订数据实际映射到文本渲染层。
2. 格式修改修订类型（FORMAT）从未被触发。虽然常量文件和 CSS 中已定义了该类型，但工具栏缺少加粗、斜体等格式化按钮，handleParagraphChange 中也只判断了新增和删除两种类型。需要增加格式化工具栏按钮并在修订模式下生成对应的 FORMAT 修订记录。
3. 批注的文本选中功能被替换成了下拉框选择段落——用户无法通过鼠标在编辑区划选文字来创建批注，评论卡片中的原文引用始终是段落前 30 个字而非用户实际选中的文字。需要恢复基于 selectionchange 事件的划选文字创建批注功能。
4. 文档编辑区中光标位置的渲染不准确。当前使用字符索引乘以固定系数的方式映射为像素位置，与实际文字位置无关联。此外光标 span 元素混入 contentEditable 区域会干扰用户编辑。需要改为用独立的叠加层配合 Range API 获取字符真实 DOM 坐标来渲染光标。
5. 删除类型的修订记录内容与实际删除位置不匹配。当用户在段落中间删除文字时，被标记为删除的是段落末尾的文字而非实际被删除的部分。需要用 diff 算法精确识别删除位置。
6. 协作光标的移动间隔不符合"每隔 3-8 秒随机移动"的要求。当前 setInterval 的间隔在首次执行时就固定为某个具体数值不再变化。需要用 setTimeout 递归调用实现每轮重新随机。
7. 运行 pnpm run test，需要让所有测试通过。
