你实现的多人协作文档编辑模拟页面在上一轮修复后存在两个回归问题，需要进行修复：

1. 在修订模式下选中文字点击格式化按钮（如加粗）时，修订面板中同时出现了 FORMAT 类型修订记录和多条 ADD 类型修订记录。多条 ADD 记录对应的是格式化过程中注入到段落文本的 markdown 标记符，这些标记符属于内部实现细节，不应作为内容变更向用户展示。
2. 在非修订模式下点击格式化按钮后，markdown 标记符（如 `**`、`*`）被直接写入了段落内容中。由于非修订模式下文本不经过修订样式渲染，用户看到的是带星号的原生文本而非粗体或斜体的视觉效果。需要让格式化在非修订模式下也能呈现用户可见的文本样式变化。
3. 运行 pnpm run test，需要让所有测试通过。
