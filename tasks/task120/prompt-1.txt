在当前项目中实现网络请求拦截器调试工具页面，所有拦截规则和日志数据在前端维护，支持 localStorage 持久化。

这个页面模拟一个网络请求拦截与 Mock 调试工具，用户可以配置请求和响应的拦截规则、编辑 Mock 数据、查看请求日志时间线。

1. 请求与响应拦截规则配置：页面分为请求拦截和响应拦截两个分区。每个分区展示拦截规则列表——每条规则包含规则名称、匹配 URL 模式（支持通配符 "*" 和精确匹配，例如 "/api/user*" 或 "/api/login"）、启用开关。请求拦截规则可配置"修改请求头"——以键值对表格编辑要添加或覆盖的请求头（如 Authorization、Content-Type）。响应拦截规则可配置"修改响应体"——在文本编辑框中输入要替换的响应 JSON 数据。提供"新增规则"按钮——弹出表单填写规则名称、URL 模式、修改内容。规则按匹配优先级排序。
2. 规则排序与开关：规则列表支持拖拽排序——拖拽调整规则在列表中的顺序，越靠上的规则优先级越高。每条规则有独立的启用/禁用开关——关闭后该规则不生效但保留配置。提供"全部启用"和"全部禁用"批量操作按钮。规则 URL 匹配从列表顶部开始匹配，第一个匹配到的规则生效（请求拦截和响应拦截分别独立匹配）。提供"测试匹配"功能——输入一个 URL 后显示哪些规则会匹配到该 URL（高亮匹配的规则行）。
3. Mock 数据 JSON 编辑：页面提供 Mock 数据编辑区——一个分栏的 JSON 编辑器，左侧为原始响应数据（模拟服务器返回的原始 JSON），右侧为 Mock 替换数据。编辑器支持 JSON 格式化（一键美化缩进）、JSON 校验（语法错误时行号高亮红色提示）。Mock 数据可保存为预设模板——给模板命名并保存，之后可从模板列表中加载。提供"Mock 数据快速填充"——预设几种常见的 API 响应模板（如用户列表、商品详情、分页数据）供快速选择填充。
4. 请求日志时间线：页面下方显示请求日志时间线——以时间线列表展示所有被拦截的模拟请求。每条日志包含请求时间（精确到毫秒）、请求方法（GET/POST/PUT/DELETE 以不同颜色标签标记）、请求 URL、状态码（200 绿色、400 黄色、500 红色）、请求耗时（毫秒）、是否被拦截规则命中（命中标记"已拦截"标签）。日志按时间倒序排列，最新请求显示在最上方。日志列表支持按请求方法、状态码、是否被拦截进行筛选。提供"清空日志"按钮。日志区域自动滚动到最新条目。
5. 模拟请求发送：页面顶部提供"发送测试请求"功能区——包含请求方法下拉选择（GET/POST/PUT/DELETE）、URL 输入框、请求体 JSON 编辑区（POST/PUT 时显示）。点击"发送"按钮后模拟发送一个网络请求——请求经过拦截规则链（先匹配请求拦截规则修改请求头，再模拟服务器响应，再匹配响应拦截规则修改响应体）。请求完成后在日志时间线中新增一条日志记录。响应详情可展开查看——显示最终请求头、请求体、响应头、响应体、原始响应（拦截前）和最终响应（拦截后）的对比。响应体以 JSON 格式化高亮展示。
6. 拦截规则导入导出：提供"导出规则"按钮——将所有拦截规则（包括请求拦截和响应拦截规则、Mock 数据模板）序列化为 JSON 文件下载。提供"导入规则"按钮——选择 JSON 文件将规则导入，导入前预览确认——展示将要导入的规则数量和规则名称列表，确认后覆盖当前规则。导入时进行 JSON 格式校验，格式错误给出提示。
7. 代码实现需要放在 src/pages/network-interceptor/ 下，测试代码放在 src/pages/__tests__/network-interceptor/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑（如 URL 通配符匹配算法、拦截规则优先级排序、请求/响应修改合并逻辑、日志时间线过滤、JSON 校验与格式化等）。
