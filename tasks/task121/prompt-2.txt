你实现的颜色工具集页面还存在一些问题，需要进行修复：

1. brandColorExtractor.js 中的颜色量化算法存在问题：量化因子和合并相似颜色的阈值均为固定魔术数字且过大，处理颜色丰富的品牌图片时会将视觉上可区分的主色错误合并，导致提取不到完整的 5 种品牌色。需要让颜色提取逻辑能根据图片实际色彩分布自适应，对相近的颜色应做加权平均而非简单丢弃。

2. index.jsx 中色盲模拟的 Canvas 渲染布局不合理：原色色块宽度占了一半 Canvas，三个模拟色块挤在剩下半区各自只占六分之一宽度，原色与模拟色比例严重失衡，用户难以公平对比颜色差异。应将 Canvas 宽度四等分，原色和三种模拟色各占一份。

3. index.jsx 的品牌色提取展示区域中，每个品牌色色块上显示的是排名数字 1~5 而非 HEX 颜色值，用户需要将视线下移到独立的文字区域才能看到颜色代码。色块应直接显示 HEX 值，保持与调色板、格式互转等其他工具一致的交互方式。

4. gradientGenerator.js 的 generateLinearGradient 函数在传入非法方向字符串时不报错也不回退到默认方向，直接拼入无效 CSS 导致预览区静默空白。应增加兜底校验，非法方向时回退到安全默认值并给出提示。

5. 修复后为该页面补充对应纯 JS 函数的单元测试，覆盖以上修复逻辑。确保 pnpm run test 中所有测试通过。
