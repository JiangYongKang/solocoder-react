在当前项目中实现「组织架构图编辑器」页面，使用 localStorage 做数据持久化。

这个页面是一个可视化的组织架构图编辑工具，用户可以创建和管理包含部门、职位、人员的树形组织架构，支持节点拖拽增删、布局切换、缩放平移、属性编辑和数据导入导出。

1. 画布中展示树形组织架构图，节点类型分为三种：部门节点（企业组织）、职位节点（总监/经理/工程师等）、人员节点（具体员工姓名）。三种节点在视觉上需要有明显区分（形状或颜色不同）。
2. 左侧提供节点模板面板，包含三种节点类型，从面板拖拽节点到画布中目标节点上即可为其添加子节点。画布中的节点支持拖拽重新排序（同级之间），也可以拖拽改变父子关系。右键点击节点弹出菜单提供编辑和删除操作，删除部门节点时需提示「该部门下包含 X 个子节点，确认删除？」。
3. 顶部工具栏提供「横向布局」和「纵向布局」两种切换按钮。横向布局——根节点在左侧，子节点向右展开；纵向布局——根节点在顶部，子节点向下展开。切换时保持选中状态，布局实时更新。
4. 画布支持鼠标滚轮缩放（范围 30%-200%）和拖拽平移。右下角显示当前缩放比例，并提供「适应画布」按钮一键居中显示全部节点。
5. 点击任意节点后，弹出属性编辑面板，可修改节点名称、节点类型（部门/职位/人员），人员节点额外可编辑邮箱和手机号。属性修改后实时更新到画布和树结构数据中。
6. 顶部工具栏提供「导入 JSON」和「导出 JSON」按钮——将当前组织架构数据序列化为 JSON 文件下载，也支持从本地 JSON 文件导入还原架构图。同时提供「导出 PNG」按钮——将画布当前可见区域导出为图片下载。
7. 代码实现需要放在 src/pages/org-chart/ 下，测试代码放在 src/pages/__tests__/org-chart/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑（如树形结构的增删改查、层级遍历算法、JSON 序列化与反序列化校验、布局方向计算、节点类型切换逻辑等）。
