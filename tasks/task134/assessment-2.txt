产物不满意：
1. 模型在 generateCSS 里用 hasCustomStyle 判断为非合并单元格生成独立 CSS 选择器时，条件包含了边框、对齐等全部属性。当用户只改对齐不改边框时，该单元格也被标记为 hasCustomStyle 并生成独立规则块，但输出的是和默认值一样的边框声明，产生了冗余代码。原因是判断范围过于宽泛。应该收窄为仅当边框颜色、粗细、样式至少有一项与默认值不同时才需要独立选择器。
2. MergeDialog 的 endRow 参数已补传，行坐标计算已修正。generateHTML 和 generateCSS 中的差异化选择器使两个不同边框色的 1x1 单元格能各自保留独立样式。代码预览已从独立弹窗移至右侧属性面板底部，以 HTML 和 CSS 标签切换实时展示。93 个测试全部通过。
