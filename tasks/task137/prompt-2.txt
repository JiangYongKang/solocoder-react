你实现的短信模板编辑器还存在一些问题，需要进行修复：

1. 模板编辑器中的变量占位符在编辑区中没有蓝色高亮标记：提示词要求"变量名在编辑区中以蓝色高亮背景标记"，但编辑区使用了原生 textarea，无法显示 HTML 样式的高亮效果。变量高亮仅在右侧预览面板中生效，编辑区中用户无法直观区分哪些是变量占位符、哪些是普通文本。

2. buildPreviewContent 函数中对签名的空值判断存在缺陷：当调用方传入空字符串作为签名值时，由于 `||` 运算符将空字符串视为 falsy，函数会回退到默认签名，导致默认场景下预览内容出现两个重复的签名前缀。

3. 首页的任务卡片列表中缺少短信模板编辑器的入口卡片，用户无法从首页点击进入该页面。

4. preview 面板的按钮使用了非标准 HTML 属性 `size="small"`，React 会将其作为自定义属性传递给底层 DOM 元素。

5. 修复完成后重新运行 pnpm run test，需要让所有测试通过。
