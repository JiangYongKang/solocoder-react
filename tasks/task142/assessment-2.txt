产物不满意：
1. index.jsx进度条虽不再用Math.random，但displayCurrentPeople用单团进度反推人数而非聚合真实总数。getProductGroupStats内部算好了aggregateCurrentPeople和totalJoinedPeople，JSX中完全没用。提示词要求"聚合real数据"，当前只反映某一个团。正确做法是用aggregateCurrentPeople做显示值、aggregateTotalPeople做分母、averageProgress驱动进度条。
2. index.jsx 团数量标签用 `productStats.totalGroups` 显示"个团进行中"，但 totalGroups 来自 `productGroups.length` 的全量统计，包含已成功和已失败的团，并非进行中的团数量。当该商品下1个成功、1个失败、1个进行中时，标签会错误显示"3个团进行中"，实际活跃的只有1个。看到这个数字会误以为有大量活跃团可参与但多数已结束。正确做法是将 totalGroups 替换为 ongoingGroups，已经算好了无需额外逻辑。
3. utils.js 的 getProductGroupStats 计算 bestProgress 时遍历了所有团包括已失败的，未用 isGroupFailed 过滤。失败团超时前 currentPeople 较高的其进度可能高过进行中的团，bestGroup 指向已失败团数据无效。进度条错误展示失败团填充宽度，与团列表状态矛盾。正确做法是在 forEach 前先过滤掉 isGroupFailed 为 true 的团。

过程不满意：
模型完成功能修复和测试后运行 lint，发现大量 React Compiler lint 错误：activityEndTime 用 useRef 但应为 state、未使用的回调函数、useEffect 同步调 setState 触发规则。模型进行了三轮以上逐批修复，第一次对 activityEndTime 的编辑还因匹配失败被迫重读。这些反复修改说明编写初始代码时没考虑 React Compiler lint 规则，没在提交前自查。正确做法是每次大修改后立即 lint 自检。
