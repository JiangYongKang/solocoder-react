产物不满意：
1. index.jsx 中 useEffect 每秒更新 registerTime 导致登记时间字段像时钟一样跳动。提示词要求"自动获取当前时间并显示，不可修改"，语义是初始化时填充一次而非实时走动。且 handleSubmit 用 Date.now() 记录登记时间，与表单跳动显示的值不同步，看到的是 23:12:33 提交时记录的可能是 23:12:34。正确做法是去掉定时器，仅在初始化时设置一次。
2. index.jsx 中 `setRecords((prev) => [...prev])` 的 60 秒定时器通过创建新数组引用触发重渲染来刷新超时判定，但副作用大：每次创建新引用都触发 useEffect 执行 saveRecords 向 localStorage 全量写入，每分钟一次不必要的持久化 I/O。应采用更轻量的方式刷新状态，如用独立的计数器 state 通过它触发 useMemo 重算，避免直接修改 records 引发连锁持久化写入。

过程不满意：
1. 模型首次编写完 index.jsx 后运行 ESLint，终端输出大量语法错误（syntax errors），模型没有读取 lint 输出中的具体错误行号和错误类型，也没有逐处排查修复，而是直接选择重写整个约 890 行的 index.jsx 文件。这种"推倒重来"的试错式修复方式浪费了一轮开发时间，且重写过程仍可能引入新的语法问题。正确做法是先通读 lint 报告的每个错误，定位到具体行号，做最小范围的字符串修复，再重新运行 lint 确认修复成功后再继续。
2. 模型首次创建 validators.js 时使用了 CommonJS 的 `require('./constants')` 语法引入依赖，而项目中所有已有文件均使用 ES Module 的 `import/export` 规范。模型在写完文件后才意识到规范不符并修改为 import 语法，暴露出模型在创建新文件前未充分核实项目已有模块导入规范的问题。应先用 Grep 搜索项目中的 import 模式确认规范，再按照统一风格编写新文件。
3. validators.js 中 maskIdCard 首次实现时 slice 参数写错，对18位身份证中间8位掩码长度和位置不符提示词要求。模型写完没核对就提交，等测试失败才被动修复。编写工具函数时应在落笔阶段对照提示词逐字验证，不该靠测试兜底。
4. 模型运行测试命令时反复出现参数格式错误，如带 `--run` 的多余参数，均因错误参数导致失败，正确命令为 `pnpm test src/pages/__tests__/visitor-registration/`。模型经过两次失败才找到正确格式，说明对 vitest CLI 参数不熟悉，应在首次失败时先用 `pnpm test --help` 核验参数格式再重试。
