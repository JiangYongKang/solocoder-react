产物不满意：
1. handleTimeout 超时释放座位后调了 setSelectedIds([])，但没重置 manualFallback 为 false。双人模式下手动选座后超时，重新选座时 manualFallback 仍为 true，handleSeatClick 直接走单人模式跳过自动匹配相邻座位的入口。根因是 timeout 清理忽略了 manualFallback 状态。正确做法是在 setSelectedIds([]) 之后增加 setManualFallback(false)。
2. 手动降级路径中，handleSeatClick 在 manualFallback 为 true 时没对已选数量做上限校验。双人模式下可选中3个以上座位，与"每次选2个座位"语义不一致。根因是复用 toggleSeatSelection 时没包装人数上限校验。正确做法是在 manualFallback 分支检查 selectedIds.length 是否达 personCount，超限时阻止继续添加。
3. handleConfirm 中 setLockedIds 用了函数式更新确保获取最新 lockedIds，但 saveSeatState 用了闭包中的旧 lockedIds 构建保存数据。虽 Set 去重后新旧数据碰巧相同，两者数据来源不一致构成潜在数据竞争。未来合并逻辑变更时，持久化的 lockedIds 会丢失已确认的锁定记录。正确做法是将合并结果提取为局部变量，setLockedIds 和 saveSeatState 共用同一份数据。
过程不满意：
1. 模型编写 updatePriceZoneConfig 时，首次将 zone key 硬编码为大写 'VIP' 直接访问 PRICE_ZONE_CONFIG，没先确认实际 key 为小写 'vip'，导致测试报错。看到失败后才回查并用 toUpperCase() 修复。写代码前没梳理 zone key 从入参到常量 key 的完整映射链路，属于试错式开发。正确做法是先读 constants.js 确认大小写约定再落笔。
2. 模型修复 set-state-in-effect 警告时，将 useEffect 内调 loadSeatState 并 setState 的模式替换为 lazy initializer 加拆分多个 useState，还移除了初始化 useEffect。这额外引入状态初始化架构变更，增大改动范围和回归风险。对 localStorage 初始化场景，更小风险的修复是加 eslint-disable 注释或保留原结构做最小改动。
3. 模型首次运行 lint 时传文件路径给 pnpm lint 但不支持，输出了大量其他包错误，随后改用 npx eslint 精准检查。运行校验前没确认 lint 命令实际行为，浪费了一轮扫描。正确做法是先查看 package.json 中 lint scripts 定义再执行。
