在当前项目中实现「命令行终端模拟」页面。该页面是首页任务卡片「命令行终端模拟」点击后的二级页面，纯前端模拟，不依赖后端服务。

这个页面模拟一个类 Unix 的命令行终端界面，支持基本的文件系统操作和命令交互。

1. 终端界面渲染：页面主体是一个深色背景的命令行终端窗口，顶部有模拟的终端标题栏（显示 terminal 字样）。终端内部显示命令行提示符（如 user@host:~$），光标闪烁，用户在此输入命令。输入的命令和输出的结果都在终端窗口中滚动显示。
2. 文件系统模拟：内置一个模拟的文件系统，初始包含多层级的目录和文件（如 /home/user/documents/、/home/user/downloads/、/etc/config/ 等），以及一些模拟的文本文件。支持以下命令：
   - ls：列出当前目录下的文件和文件夹，文件夹用不同颜色标识
   - cd <目录>：切换当前工作目录，支持 cd .. 返回上级和 cd / 回到根目录
   - cat <文件名>：显示指定文件的内容
   - mkdir <目录名>：在当前目录下创建新文件夹
   - pwd：显示当前工作目录的完整路径
   - 对于不支持的命令，输出类似 bash 的报错信息（如 command not found: xxx）
3. 历史命令回显：按键盘上键可以回显上一条输入过的命令，按下键可以回显下一条。命令历史在会话期间保持。
4. 自动补全提示：按 Tab 键触发自动补全——如果当前输入的前缀能唯一匹配一个文件或目录名，则自动补全；如果有多个匹配，则在输入下方显示所有匹配项列表。
5. 命令高亮：输入的命令按语法高亮——命令本身用一种颜色（如绿色），参数用另一种颜色（如白色），路径用第三种颜色（如蓝色）。高亮在用户输入过程中实时渲染。
6. 清屏功能：输入 clear 命令后清空终端窗口中的所有输出内容，只保留提示符。也可以提供一个清屏按钮。
7. 代码实现需要放在 src/pages/terminal-simulator/ 下，测试代码放在 src/pages/__tests__/terminal-simulator/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑（如文件系统路径解析、命令解析、自动补全匹配等）。
