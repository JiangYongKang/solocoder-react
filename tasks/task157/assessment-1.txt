产物不满意：
1. abTestCore.js 导出了 regularizedIncompleteBeta、betaCF、gammaLn 三个完整统计函数（约 100 行），但 calculatePValue 并未调用它们，而是用手写分段线性近似算 p 值。这三个函数完全不被引用，且内部还有嵌套调用链，增加了维护负担。原因是先实现了 Beta 分布精确算法但测试不通过，改用分段近似后忘了清理旧代码。应移除未使用函数或改造 calculatePValue 调用它们。
2. updateTimeSeriesData 执行逻辑是 [...currentData.slice(0, -1), lastEntry]，即每次定时器触发只替换最后一天数据点，而非在末尾追加新一天。无论实验跑多久，图表始终只有 7 天数据，实时刷新完全体现不了，第四个需求"数据用定时器模拟周期性刷新"效果大打折扣。原因是混淆了"更新最新点"和"追加新点"的概念。应追加新条目而非替换旧条目。
3. constants.js 中定义了 `METRIC_KEYS` 并导出，但搜索整个 ab-test 包的所有文件，该常量从未被任何地方 import 或引用。这是一个无实际用途的冗余导出，且该常量仅是对 `METRICS` 数组做了一次 `.map((m) => m.key)` 的派生结果，即使未来需要也应在调用处实时计算而非预先导出全局常量。原因是编写 constants.js 时顺手定义了辅助变量，但后续开发中未找到使用场景也未清理。
4. index.jsx 中 ExperimentDetail 的 useEffect(() => { setCurrentExperiment(experiment) }, [experiment]) 是冗余。currentExperiment 已由 useState(experiment) 设初值，父组件传新 experiment 时自然重挂载。这个 effect 多余且引发 set-state-in-effect。应删除，需同步 prop 时用 key={experiment.id} 重建。

过程不满意：
