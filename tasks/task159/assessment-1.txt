产物不满意：
1. DanmakuArea.jsx 第一个 useEffect 用 for 循环创建 6 个 setTimeout 发初始弹幕，cleanup 没有 clearTimeout。快速进出直播间时，卸载后回调仍 addDanmaku 触发 setState，React 报 unmounted 警告。原因是只关注启动忽略清理，模型重写时仍遗漏。应收集 timeout ID 在 cleanup 中遍历清除。
2. index.jsx 给 ShoppingBag 加了 key={currentProduct?.id} 使商品切换时销毁重建，selectedSpecs、addQuantity 等状态被重置。轮播每 3 秒自动切换时，如正在浏览规格未点击加入，临时选择静默丢弃。用 key 强制 remount 规避 ESLint Hooks 警告，而非在 ShoppingBag 内监听变化重置，牺牲了交互连续性。
3. LikeButton 的 handleClick 计算爱心位置有坐标偏移。代码通过 offsetParent 获取容器坐标，按钮中心减容器左上角得 x、y，定位在 hearts-layer。但 hearts-layer 的 CSS 设了 inset:-300px -100px 0 -100px，该层原点比容器左上角偏左 100px 偏上 300px。爱心出现位置与按钮相差约 100px/300px，并非从正中心飘出。应 inset 统一为 0，或对 x、y 加偏移补偿。
过程不满意：
1. 模型完成所有组件和 CSS 后才首次跑 ESLint，发现大量 JSX style 对象像素值未用引号包裹的语法错误，如 {width:12px} 应为 {width:'12px'}。错误分布在多文件，导致整体重写 DanmakuArea.jsx 和 ShoppingBag.jsx，重写后 ShoppingBag 又出现 Hooks 依赖问题需第三轮修改。若每完成一个组件就跑 lint，这些错误会在第一时间修复，不会积累到全文替换的程度。
2. 模型遇到 ESLint Hooks exhaust-deps 警告时，直接选了 key={currentProduct?.id} 让 ShoppingBag 强制 remount。思考过程只记录了"用 key 是更好的方式"，没分析此方案会导致轮播自动切换时组件内部状态丢失的副作用，也没讨论 useEffect 监听 currentProduct 变化的替代方案。这反映面对 lint 警告时倾向选最快消除的捷径，而非深入分析根因和评估方案对交互体验的影响。
3. DanmakuArea.jsx 第一个 useEffect 用了 setTimeout 但 cleanup 为空的问题，在模型重写过程中始终没被发现和修复。首次编写和整文件重写时都只关注启动弹幕逻辑，没遵循 React 副作用清理的对称性原则为 setTimeout 添 clearTimeout。说明对 useEffect cleanup 机制理解不深，没形成"每个需清理的副作用都要有对应清理代码"的编码习惯。
