在当前项目中实现「任务依赖图」页面。该页面是首页任务卡片「任务依赖图」点击后的二级页面，使用 Canvas 或 SVG 渲染有向无环图，数据持久化到 localStorage。

这个页面是一个任务依赖关系的可视化编辑器，帮助用户构建和管理任务之间的前置依赖关系。

1. 节点拖拽：页面主体是一个可缩放平移的画布。用户可以点击画布空白区域或通过「添加节点」按钮创建新的任务节点。每个节点显示任务名称，节点可在画布上自由拖拽移动位置。双击节点可编辑任务名称。
2. 连线建立依赖：从一个节点的输出端口（右侧或底部小圆点）拖拽到另一个节点的输入端口（左侧或顶部小圆点），即可建立一条有向连线，表示前者是后者的前置依赖。连线用带箭头的直线或贝塞尔曲线绘制。当尝试建立会形成环路的连线时（违反有向无环图约束），给出提示并拒绝操作。
3. 拓扑排序展示：提供一个「拓扑排序」按钮，点击后对当前图中所有节点计算拓扑排序，并将排序结果按顺序展示在画布旁边或弹窗中——展示按依赖顺序排列的任务列表（排在前面的任务可以先执行，排在后面的任务需要等待前置任务完成）。
4. 关键路径高亮：提供一个「关键路径」按钮，点击后计算从没有前置依赖的起始节点到没有后继依赖的终止节点之间的最长路径（关键路径），并将关键路径上的节点和连线用特殊颜色（如红色）高亮显示。
5. 缩放平移：画布支持鼠标滚轮缩放和按住空白区域拖拽平移。画布右下角提供缩放比例指示器和「适应画布」按钮（将所有节点缩放到可视区域内）。
6. JSON 导入导出：提供「导出」按钮，将当前图中的所有节点（id、名称、位置坐标）和所有连线（起始节点 id、目标节点 id）导出为 JSON 文件下载。提供「导入」按钮，从 JSON 文件读取数据还原图结构。刷新页面时从 localStorage 恢复上次的图状态。
7. 自动布局：提供一个「自动布局」按钮，点击后用层次布局算法自动排列所有节点——依赖关系前置的节点在上方或左侧，后续节点在下方或右侧，节点间距均匀，连线避免重叠。
8. 代码实现需要放在 src/pages/task-dag/ 下，测试代码放在 src/pages/__tests__/task-dag/ 下。还需要为该页面编写纯 JS 函数的单元测试，只测和页面渲染无关的逻辑（如图数据结构、环路检测、拓扑排序、关键路径计算、自动布局算法等）。
